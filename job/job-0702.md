### 🟡 PM-94：綠界 ECPay 測試 key 換正式 key（從 wrangler.toml 遷移到 secret）

**背景**：綠界信用卡 + 非信用卡收款皆審核通過（2026-07-02）。ECPay key 目前寫死在 `wrangler.toml`（測試值），需遷移到 `wrangler secret` 並切換正式環境。

**目標**：測試環境 → 正式環境，key 從明文改為 secret。

---

#### §1 從 wrangler.toml 刪除 ECPay 四行

刪除 `server/wrangler.toml` 中的：
```
ECPAY_MERCHANT_ID = "3002607"
ECPAY_HASH_KEY = "pwFHCqoQZGmho4w6"
ECPAY_HASH_IV = "EkRm7iFT261dpevs"
ECPAY_PAYMENT_URL = "https://payment-stage.ecpay.com.tw/Cashier/AioCheckOut/V5"
```

---

#### §2 先 deploy 一次清掉舊 binding

```bash
cd server && npx wrangler deploy
```

這步是為了讓 Cloudflare 清除 wrangler.toml 裡的舊 binding，之後才能用同名的 secret。

---

#### §3 用 wrangler secret put 存正式 key

```bash
npx wrangler secret put ECPAY_MERCHANT_ID
# 貼入：3505501

npx wrangler secret put ECPAY_HASH_KEY
# 貼入：5tnieA8TAmuPsY13

npx wrangler secret put ECPAY_HASH_IV
# 貼入：HvQbyLgYRick9zDl

npx wrangler secret put ECPAY_PAYMENT_URL
# 貼入：https://payment.ecpay.com.tw/Cashier/AioCheckOut/V5
```

---

#### §4 server/src/index.ts 確認 Env 型別

確認 Env interface 有這四個欄位（PM-94 原始規格已列）：
```ts
ECPAY_MERCHANT_ID: string;
ECPAY_HASH_KEY: string;
ECPAY_HASH_IV: string;
ECPAY_PAYMENT_URL: string;
```

確認程式碼中所有 ECPay 相關邏輯都用 `env.ECPAY_*` 讀取，沒有寫死測試值。

---

#### §5 搜尋殘留的測試值

```bash
grep -rn "3002607\|pwFHCqoQZGmho\|EkRm7iFT261\|payment-stage" server/src/
```

如有殘留，全部改成讀 env。

---

#### §6 再 deploy 一次

```bash
npx wrangler deploy
```

---

#### §7 不改的東西

- extension / CLI / MCP：不動
- Supabase：不動
- 首頁 / guide / faq / privacy：不動

#### 驗收

1. 開無痕視窗 → 登入 BugEzy → 按升級付費
2. 應導向**正式**綠界付款頁（`payment.ecpay.com.tw` 不是 `payment-stage`）
3. `grep -rn "3002607\|payment-stage" server/` 應無結果

### 🟡 PM-108：首頁「立即升級」按鈕導向安裝引導

**問題**：首頁定價區的「立即升級」按鈕目前無法直接付款（首頁是公開網頁，沒有擴充功能的付款流程）。需要引導使用者先安裝，再從 popup 升級。

**目標**：「立即升級」按鈕導到 `/install` 頁，附帶提示文字。

---

#### §1 首頁 HTML：改「立即升級」按鈕

現有：`<a href="#" ...>立即升級</a>` 或類似

改為：
```html
<a href="/install" class="upgrade-btn">安裝後即可升級 →</a>
```

按鈕文字從「立即升級」改成「安裝後即可升級 →」，讓使用者預期正確。

---

#### §2 付費版卡片加一行提示

在付費版的功能列表下方、按鈕上方加一行小字：

```html
<p class="pricing-hint">安裝 Chrome 擴充後，在工具中一鍵升級付費</p>
```

CSS：
```css
.pricing-hint {
  color: #9aa3b2;
  font-size: 12px;
  margin-top: 12px;
  margin-bottom: 8px;
}
```

---

#### §3 免費版卡片也加按鈕

免費版目前沒有按鈕。加一個：

```html
<a href="/install" class="free-btn">免費安裝 →</a>
```

樣式比付費版低調（透明底 + 邊框）。

---

#### §4 不改的東西

- extension / MCP / Supabase：不動
- popup 升級付費邏輯：不動
- /install、/features、/guide 頁面：不動

#### Deploy

```
cd server && npx wrangler deploy
```

#### 驗收

1. 首頁定價區 → 付費版按鈕應顯示「安裝後即可升級 →」→ 點擊導到 `/install`
2. 付費版卡片有提示文字
3. 免費版卡片有「免費安裝 →」按鈕 → 點擊導到 `/install`
