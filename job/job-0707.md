### ✅ PM-187：修復 /reports 等頁面 URL token 洩漏資安問題（已完成）

**背景**：目前 `/reports` 頁面將 session token 放在 URL query string（`/reports?token=xxx`），造成三個資安風險：
1. **瀏覽器歷史紀錄洩漏**——任何碰到電腦的人可從 history 取得 token
2. **HTTP Referrer 洩漏**——頁面若有外連，Referrer header 會帶完整 URL（含 token）送到第三方
3. **截圖/分享洩漏**——用戶分享螢幕或截圖時 token 直接露出

**優先級**：🔴 P0（資安問題，阻塞公開推廣）

---

#### §1 Extension popup：改用 localStorage 傳 token

目前 popup 開啟 `/reports?token=xxx`，改為：

1. popup 開啟「我的報告」時，先用 `postMessage` 或 URL fragment 將 token 傳給目標頁
2. **推薦做法**：popup 開啟乾淨的 `bugezy.dev/reports`（無 query string），頁面載入後用 `window.postMessage` 或 `BroadcastChannel` 從 extension 傳 token
3. **備選做法**：用 URL fragment `bugezy.dev/reports#token=xxx`（fragment 不送 server、不出現在 Referrer），頁面讀完後立即 `history.replaceState` 清除 fragment

> 注意：跨域 localStorage（extension vs bugezy.dev）不共享，所以 popup 不能直接 set localStorage 給 bugezy.dev。需選用可行的跨域傳遞方式。

---

#### §2 Server /reports 頁面：讀 token 後清除 URL

```
頁面載入邏輯（優先順序）：
1. 檢查 localStorage('bugezy_session_token')
   → 有：直接用，URL 保持乾淨
2. 檢查 URL query string ?token=
   → 有：讀取 → 存入 localStorage → history.replaceState 清除 URL 中的 token
3. 檢查 URL fragment #token=
   → 有：讀取 → 存入 localStorage → history.replaceState 清除 fragment
4. 以上皆無：
   → 顯示「請先從 BugEzy 擴充登入」提示
```

### ✅ PM-188：報告分享閱讀權限——非擁有者需付費會員才能讀（已完成）

**背景**：目前 `/report/:id` 是「有連結就能看」，任何人拿到 UUID 連結就能讀取完整報告內容（含 console logs、network errors、語音描述等），有敏感資料洩漏風險。若使用者將報告連結貼在公開論壇，所有人都能看到內容。

**商業價值**：此功能同時製造付費動機——別人分享 BugEzy 報告連結，想看的人必須成為付費會員，形成病毒式傳播 → 引導付費。

**優先級**：🔴 P0（資安 + 商業雙重理由）

---

#### §1 核心規則

```
請求 GET /api/reports/:id 時：

1. 從 request header 讀 Authorization: Bearer <token>（可選）
2. 若有 token → verifySession 取得 userId
3. 查 report 的 owner_id

判斷邏輯：
├── 無 token（訪客）→ 403「請登入 BugEzy 並升級會員才能閱讀」
├── 有 token，userId === report.owner_id → ✅ 放行（本人看自己的，不管付費狀態）
├── 有 token，userId !== owner_id → 檢查付費狀態
│   ├── 是付費會員（day_pass 未過期 或 subscription 有效）→ ✅ 放行
│   └── 免費用戶 → 403「升級會員即可閱讀他人分享的報告」
```

---

#### §2 Server 端 getReport() 改動

目前 `getReport()` 無認證直接回報告，需改為：

```ts
async function getReport(reportId: string, request: Request, env: Env): Promise<Response> {
  // 1. 查報告（先確認存在）
  const report = await queryReport(reportId, env);
  if (!report) return json({ error: 'report not found' }, 404);

  // 2. 嘗試讀 token（可選，不強制）
  const authHeader = request.headers.get('Authorization');
  const token = authHeader?.startsWith('Bearer ') ? authHeader.slice(7) : null;

  let userId: string | null = null;
  if (token) {
    const session = await verifySessionByToken(token, env);
    if (session) userId = session.user_id;
  }

  // 3. 權限判斷
  const isOwner = userId && userId === report.user_id;

  if (!isOwner) {
    // 非擁有者 → 必須是付費會員
    if (!userId) {
      // 訪客（無 token）
      return json({ error: 'login_required', message: '請登入 BugEzy 並升級會員才能閱讀' }, 403);
    }
    // 有登入但不是 owner → 檢查付費
    const isPaid = await checkPaidStatus(userId, env);
    if (!isPaid) {
      return json({ error: 'upgrade_required', message: '升級會員即可閱讀他人分享的報告' }, 403);
    }
  }

  // 4. 通過 → 回傳報告（原本邏輯不動）
  // ... 原本的 R2 取 rrweb / screenshots 邏輯 ...
}
```

#### §3 checkPaidStatus() 共用函式

```ts
// 檢查使用者是否為付費會員（day_pass 未過期 或 subscription 有效）
async function checkPaidStatus(userId: string, env: Env): Promise<boolean> {
  const { data } = await supa(env)
    .from('payments')
    .select('plan, expires_at')
    .eq('user_id', userId)
    .order('expires_at', { ascending: false })
    .limit(1)
    .single();

  if (!data) return false;
  return new Date(data.expires_at) > new Date();
}
```

```
3. 若是付費會員 → 先顯示免責警語彈窗 → 用戶確認後才執行複製/匯出
```

#### §2 免責警語（每次操作都顯示）

用戶點擊「複製 JSON」或「匯出 JSON」且確認為付費會員後，顯示確認彈窗：

```
⚠️ JSON 資料包含完整除錯紀錄

此資料可能包含敏感資訊（API 金鑰、Token、錯誤訊息等），
請勿將 JSON 資料分享給不信任的對象。

因資料外洩造成的損失，BugEzy 不承擔責任。

[取消]  [我了解，繼續]
```

- 用戶按「我了解，繼續」→ 執行複製/匯出
- 用戶按「取消」→ 不執行

> 設計選擇：每次都顯示警語，不設「不再提示」。理由：這是免責聲明，法律上每次提示更安全。
