### 🟡 PM-153：Bug 捕捉升級 — console.warn 完整捕捉（CORS #5 + Mixed Content #7）

**問題**：inject.ts 的 console 攔截可能只抓 error 層級，遺漏 warn。CORS 錯誤和 Mixed Content 警告大部分出在 `console.warn`，小白工程師最困惑的兩類問題就這樣漏掉了。

**目標**：確保 inject.ts 攔截 console.warn，且上傳報告 / 即時監控都包含 warn 層級。

---

#### §1 檢查 inject.ts console 攔截

搜尋 inject.ts 中 console 攔截的程式碼，確認是否已攔截 `console.warn`：

```ts
// 應該攔截的層級
const levels = ['error', 'warn'];

levels.forEach(level => {
  const original = console[level];
  console[level] = function(...args: any[]) {
    // 記錄到 BugEzy 收集器
    collectConsoleLog({ level, message: args.map(String).join(' '), timestamp: Date.now() });
    // 呼叫原始方法
    original.apply(console, args);
  };
});
```

如果現在只攔截 `error`，加上 `warn`。

#### §3 報告上傳確認

#### §4 MCP get_console_logs 確認

### 🟡 PM-154：Bug 捕捉升級 — unhandledrejection + window.onerror（Promise 靜默 #8 + 框架錯誤 #6）

**問題**：
- **#8 Promise 靜默失敗**：小白最常犯 `async/await` 忘了 `catch`，錯誤完全消失，console 什麼都沒有

#### §1 inject.ts：加 unhandledrejection 監聽

```ts
window.addEventListener('unhandledrejection', (event) => {
  const reason = event.reason;
  const message = reason instanceof Error
    ? `Unhandled Promise Rejection: ${reason.message}\n${reason.stack || ''}`
    : `Unhandled Promise Rejection: ${String(reason)}`;
  
  collectConsoleLog({
    level: 'error',
    message: message,
    timestamp: Date.now(),
    source: 'unhandledrejection',
  });
  
  // 即時監控也要更新
  updateMonitorCount();
});
```

---

#### §2 inject.ts：加 window.onerror 監聽

```ts
window.addEventListener('error', (event) => {
  // 區分 JS 執行錯誤 vs 資源載入錯誤
  // event.target 是 HTMLElement（img/script/link）→ 資源載入失敗（PM-155 處理）
  // event.target 是 window → JS 執行錯誤
  if (event.target === window || event.target === document) {
    const message = `${event.message || 'Script Error'} at ${event.filename || 'unknown'}:${event.lineno || 0}:${event.colno || 0}`;
    collectConsoleLog({
      level: 'error',
      message: message,
      timestamp: Date.now(),
      source: 'window.onerror',
    });
    updateMonitorCount();
  }
  // 資源載入錯誤在 PM-155 用 capture:true 處理
}, false);  // bubbling phase（JS 錯誤）
```

---

#### §3 避免重複計算

console.error 攔截 + window.onerror + unhandledrejection 可能對同一個錯誤重複記錄。加去重邏輯：

```ts
const recentErrors = new Set<string>();

function dedupeCollect(log: ConsoleLog) {
  const key = `${log.level}:${log.message.slice(0, 100)}`;
  if (recentErrors.has(key)) return; // 已記錄過，跳過
  recentErrors.add(key);
  // 5 秒後清除（允許相同錯誤再次記錄）
  setTimeout(() => recentErrors.delete(key), 5000);
  collectConsoleLog(log);
}
```

所有收集點（console 攔截 / onerror / unhandledrejection）都改呼叫 `dedupeCollect`。

#### §1 inject.ts：Resource 載入失敗捕捉

用 `window.addEventListener('error', ..., true)` 的 **capture phase** 抓資源載入錯誤（img/script/link/video 的 404 等）：

```ts
window.addEventListener('error', (event) => {
  const target = event.target as HTMLElement;
  
  // 只處理資源載入錯誤（非 JS 執行錯誤，JS 在 PM-154 的 bubbling phase 處理）
  if (target === window || target === document) return;
  
  // 取得資源 URL
  const src = (target as HTMLImageElement).src
    || (target as HTMLScriptElement).src
    || (target as HTMLLinkElement).href
    || '';
  
  if (!src) return;
  
  const tagName = target.tagName?.toLowerCase() || 'unknown';
  const message = `Resource load failed: <${tagName}> ${src}`;
  
  dedupeCollect({
    level: 'warn',
    message: message,
    timestamp: Date.now(),
    source: 'resource-error',
  });
  
  updateMonitorCount();
}, true);  // capture phase（資源錯誤不冒泡，必須 capture）
```

---

#### §2 inject.ts：Web Vitals 效能捕捉

用 `PerformanceObserver` 捕捉核心 Web Vitals：

```ts
function collectWebVitals() {
  try {
    // LCP (Largest Contentful Paint) — 最大內容繪製時間
    new PerformanceObserver((list) => {
      const entries = list.getEntries();
      const last = entries[entries.length - 1] as any;
      if (last) {
        collectPerformanceMetric('LCP', Math.round(last.startTime), 'ms');
      }
    }).observe({ type: 'largest-contentful-paint', buffered: true });

    // CLS (Cumulative Layout Shift) — 累計版面位移
    let clsValue = 0;
    new PerformanceObserver((list) => {
      for (const entry of list.getEntries() as any[]) {
        if (!entry.hadRecentInput) clsValue += entry.value;
      }
      collectPerformanceMetric('CLS', Math.round(clsValue * 1000) / 1000, '');
    }).observe({ type: 'layout-shift', buffered: true });

    // FID (First Input Delay) — 首次輸入延遲
    new PerformanceObserver((list) => {
      const entry = list.getEntries()[0] as any;
      if (entry) {
        collectPerformanceMetric('FID', Math.round(entry.processingStart - entry.startTime), 'ms');
      }
    }).observe({ type: 'first-input', buffered: true });
  } catch (e) {
    // 部分瀏覽器不支援某些 observer，靜默失敗
  }
}

function collectPerformanceMetric(name: string, value: number, unit: string) {
  // 判斷是否超標
  const thresholds: Record<string, [number, string]> = {
    'LCP': [2500, '慢（超過 2.5 秒）'],
    'CLS': [0.1, '版面位移過大'],
    'FID': [100, '互動延遲過高'],
  };
  const [threshold, desc] = thresholds[name] || [Infinity, ''];
  const status = value > threshold ? '⚠️ ' + desc : '✅ 良好';

  dedupeCollect({
    level: value > threshold ? 'warn' : 'info',
    message: `Web Vital ${name}: ${value}${unit} ${status}`,
    timestamp: Date.now(),
    source: 'web-vitals',
  });
}

// 頁面載入後執行
collectWebVitals();
```

### 🟡 PM-156：Bug 捕捉升級 — 網路環境快照

**背景**：小白常遇到「我的電腦上好的，客戶那邊就壞」。原因通常是網路環境不同——3G vs WiFi、高延遲、離線等。AI 看不到網路狀態就無法診斷。

**目標**：在錄製/截圖/即時監控上傳報告時，自動附帶網路環境快照。

---

#### §1 inject.ts：收集網路環境

```ts
function getNetworkSnapshot(): Record<string, unknown> {
  const conn = (navigator as any).connection;
  return {
    online: navigator.onLine,
    effectiveType: conn?.effectiveType || 'unknown',  // '4g', '3g', '2g', 'slow-2g'
    rtt: conn?.rtt ?? null,                            // 延遲 ms
    downlink: conn?.downlink ?? null,                  // 頻寬 Mbps
    saveData: conn?.saveData ?? false,                 // 省流量模式
    type: conn?.type || 'unknown',                     // 'wifi', 'cellular', 'ethernet'
  };
}
```

`navigator.connection` 是 Chrome 61+ 支援的 Network Information API，BugEzy 的目標瀏覽器完全沒問題。

---

#### §2 錄製時捕捉

在錄製開始時（startRecording）和結束時（stopRecording）各抓一次網路快照，存入報告 payload：

```ts
const reportPayload = {
  // ...現有欄位
  networkSnapshot: {
    atStart: getNetworkSnapshot(),
    atEnd: getNetworkSnapshot(),
  },
};
```

這樣 AI 能看到「錄製開始時 4G、結束時離線」→ 診斷 API 失敗原因。

---

#### §3 即時監控上傳也帶

PM-124 的「📤 上傳報告讓 AI 分析」也附帶網路快照。透過 postMessage 傳給 content → background：

```ts
window.postMessage({
  type: 'BUGEZY_UPLOAD_MONITOR_REPORT',
  errors: collectedErrors,
  networkSnapshot: getNetworkSnapshot(),
}, '*');
```

---

#### §4 截圖標注也帶

annotate.ts 上傳報告時也附帶 `getNetworkSnapshot()`。annotate 頁面可以直接用 navigator API（不是 MAIN world 限制的 inject.ts）。

---

#### §5 Server：createReport 接收 networkSnapshot

server 的 createReport 把 networkSnapshot 存進 Supabase reports 表。如果 reports 表沒有 networkSnapshot 欄位，加一個 JSONB 欄位：


### 🟡 PM-157：Bug 捕捉升級 — 儲存空間快照 + PII 遮罩（#9.3→9.5）

**背景**：小白常遇到「登入狀態突然消失」「資料怎麼不見了」。原因通常是 localStorage/sessionStorage/cookie 的使用問題。AI 看不到儲存狀態就無法診斷。

**敏感資料處理**：儲存空間可能含密碼、token、信用卡號。必須在**使用者端（extension）就自動遮罩**，server 永遠不碰敏感原值。

---

#### §1 inject.ts：收集儲存空間快照

```ts
function getStorageSnapshot(): Record<string, unknown> {
  return {
    localStorage: snapshotStorage(window.localStorage),
    sessionStorage: snapshotStorage(window.sessionStorage),
    cookieCount: document.cookie ? document.cookie.split(';').length : 0,
    cookieNames: document.cookie
      ? document.cookie.split(';').map(c => c.trim().split('=')[0])
      : [],
  };
}

function snapshotStorage(storage: Storage): Array<{ key: string; size: number; value: string }> {
  const items: Array<{ key: string; size: number; value: string }> = [];
  for (let i = 0; i < storage.length; i++) {
    const key = storage.key(i);
    if (!key) continue;
    const rawValue = storage.getItem(key) || '';
    items.push({
      key: key,
      size: rawValue.length,
      value: maskPII(key, rawValue),  // 遮罩敏感值
    });
  }
  return items;
}
```

#### §2 PII 遮罩（核心安全機制）

```ts
// 敏感 key 名稱模式
const SENSITIVE_KEYS = /password|passwd|pwd|token|secret|key|api.?key|auth|credential|credit.?card|card.?num|cvv|ssn|session/i;

// 敏感值模式（email、信用卡號、JWT）
const SENSITIVE_VALUES = [
  /\b[\w.-]+@[\w.-]+\.\w{2,}\b/g,                    // email
  /\b\d{4}[\s-]?\d{4}[\s-]?\d{4}[\s-]?\d{4}\b/g,    // 信用卡號
  /eyJ[\w-]+\.eyJ[\w-]+\.[\w-]+/g,                     // JWT token
];

function maskPII(key: string, value: string): string {
  // 1. key 名稱敏感 → 整個值遮罩
  if (SENSITIVE_KEYS.test(key)) {
    return '***MASKED***';
  }
  
  // 2. 值太長（>500 字元）→ 截斷 + 標示大小
  if (value.length > 500) {
    return value.slice(0, 100) + `... (${value.length} chars, truncated)`;
  }
  
  // 3. 值裡含敏感模式 → 局部遮罩
  let masked = value;
  SENSITIVE_VALUES.forEach(pattern => {
    masked = masked.replace(pattern, '***');
  });
  
  return masked;
}
```

### 🟡 PM-158：MCP 新增 `get_timeline` — 時序麵包屑（所有資料合成一條故事線）

**背景**：目前 AI 要讀一份報告需要呼叫 5+ 個 MCP tool（console logs、network errors、voice transcript、rrweb summary、report overview）。每次呼叫都花 token，而且 AI 要自己對照時間軸拼湊因果。

**目標**：新增一個 `get_timeline` MCP tool，把一份報告的所有資料按時間排序，合成一條人類可讀的時間軸。AI 只要呼叫一次就能看到完整故事。

---

#### §1 Server：新增 MCP tool `get_timeline`

```ts
server.tool(
  'get_timeline',
  '取得一份報告的完整時間軸（時序麵包屑）。把 Console、Network、語音、DOM、網路環境、儲存狀態全部按時間排序成一條故事線。AI 只需呼叫這一個 tool 就能掌握完整 Bug 脈絡。',
  {
    report_id: z.string().describe('報告 ID'),
  },
  async (args) => {
    const { report_id } = args;

    // 1. 讀取報告所有資料
    const { data: report, error } = await supa(env)
      .from('reports')
      .select('*')
      .eq('report_id', report_id)
      .maybeSingle();

    if (error || !report) {
      console.error('get_timeline failed:', error?.message);
      return txt('報告不存在或查詢失敗');
    }

    // 2. 收集所有事件到統一陣列
    const events: Array<{ time: number; type: string; icon: string; text: string }> = [];

    // Console logs
    const consoleLogs = report.console_logs || [];
    consoleLogs.forEach((log: any) => {
      const icon = log.level === 'error' ? '❌' : log.level === 'warn' ? '⚠️' : 'ℹ️';
      events.push({
        time: log.timestamp || 0,
        type: 'console',
        icon,
        text: `[Console ${log.level}] ${log.message}`,
      });
    });

    // Network errors
    const networkErrors = report.network_errors || [];
    networkErrors.forEach((net: any) => {
      events.push({
        time: net.timestamp || 0,
        type: 'network',
        icon: '🌐',
        text: `[Network] ${net.method} ${net.url} → ${net.status} (${net.duration || '?'}ms)`,
      });
    });

    // Voice transcript
    const voiceTranscript = report.voice_transcript || [];
    voiceTranscript.forEach((v: any) => {
      events.push({
        time: v.timestamp || 0,
        type: 'voice',
        icon: '🎙️',
        text: `[語音] ${v.text}`,
      });
    });

    // Markers (使用者標記的重要時間點)
    const markers = report.markers || [];
    markers.forEach((m: any) => {
      events.push({
        time: m.timestamp || 0,
        type: 'marker',
        icon: '📌',
        text: `[標記] ${m.label || 'user marker'}`,
      });
    });

    // 3. 按時間排序
    events.sort((a, b) => a.time - b.time);

    // 4. 計算相對時間（從第一個事件起算）
    const startTime = events.length > 0 ? events[0].time : 0;

    // 5. 組裝時間軸文字
    let timeline = `📋 報告時間軸 — ${report.title || report.url}\n`;
    timeline += `頁面：${report.url}\n`;
    timeline += `瀏覽器：${report.user_agent || 'unknown'}\n`;
    timeline += `螢幕：${report.screen_size || 'unknown'}\n`;

    // 網路環境
    if (report.network_snapshot) {
      const ns = report.network_snapshot;
      const atStart = ns.atStart || ns;
      timeline += `網路：${atStart.online ? '在線' : '離線'} / ${atStart.effectiveType || '?'} / RTT ${atStart.rtt ?? '?'}ms / ${atStart.downlink ?? '?'} Mbps\n`;
    }

    // 儲存快照摘要
    if (report.storage_snapshot) {
      const ss = report.storage_snapshot;
      const lsCount = ss.localStorage?.length || 0;
      const ssCount = ss.sessionStorage?.length || 0;
      timeline += `儲存：localStorage ${lsCount} 項 / sessionStorage ${ssCount} 項 / Cookie ${ss.cookieCount || 0} 個\n`;
    }

    timeline += `描述：${report.description || '（無）'}\n`;
    timeline += `─────────────────────────\n`;

    if (events.length === 0) {
      timeline += '（無事件記錄）\n';
    } else {
      events.forEach(e => {
        const relSec = ((e.time - startTime) / 1000).toFixed(1);
        timeline += `[${relSec}s] ${e.icon} ${e.text}\n`;
      });
    }

    timeline += `─────────────────────────\n`;
    timeline += `共 ${events.length} 個事件（Console ${consoleLogs.length} / Network ${networkErrors.length} / 語音 ${voiceTranscript.length} / 標記 ${markers.length}）`;

    return txtWithTokens([timeline], 'get_timeline');
  },
);
```
