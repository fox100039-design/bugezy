### 🟡 PM-153：Bug 捕捉升級 — console.warn 完整捕捉（CORS #5 + Mixed Content #7）

**問題**：inject.ts 的 console 攔截可能只抓 error 層級，遺漏 warn。CORS 錯誤和 Mixed Content 警告大部分出在 `console.warn`，小白工程師最困惑的兩類問題就這樣漏掉了。

**目標**：確保 inject.ts 攔截 console.warn，且上傳報告 / 即時監控都包含 warn 層級。

---

#### §1 檢查 inject.ts console 攔截

搜尋 inject.ts 中 console 攔截的程式碼，確認是否已攔截 `console.warn`：

```ts
// 應該攔截的層級
const levels = ['error', 'warn'];

levels.forEach(level => {
  const original = console[level];
  console[level] = function(...args: any[]) {
    // 記錄到 BugEzy 收集器
    collectConsoleLog({ level, message: args.map(String).join(' '), timestamp: Date.now() });
    // 呼叫原始方法
    original.apply(console, args);
  };
});
```

如果現在只攔截 `error`，加上 `warn`。

---

#### §2 即時監控也要捕捉 warn

即時監控的 error 偵測邏輯（inject.ts 的 `updateMonitorBadge`），如果只計算 error 不計 warn，改為兩者都計。

但 UI 要區分：
- error → ❌ 紅色
- warn → ⚠️ 橘色

error panel 顯示時用不同顏色區分。

---

#### §3 報告上傳確認

createReport payload 的 `consoleLogs` 確認包含 warn 層級的記錄，不只 error。

---

#### §4 MCP get_console_logs 確認

MCP tool 回傳的 console logs 確認包含 warn（應該本來就包含，只要 inject.ts 有收集就行）。

---

#### §5 不改的東西

- server / MCP tool 邏輯：不動（只要 inject 有收集，其他層自動帶到）
- popup / annotate：不動
- network error 捕捉：不動

#### Build

```
cd extension && npm run build
```

#### 驗收

1. 開測試頁 → 觸發 `console.warn('test warn')` → BugEzy 有捕捉到
2. 即時監控 → 觸發 warn → 狀態條顯示「⚠️ 發現 1 個錯誤」
3. 錄製 → 觸發 warn → 報告有 warn 記錄
4. MCP get_console_logs → 回傳含 warn 層級

### 🟡 PM-154：Bug 捕捉升級 — unhandledrejection + window.onerror（Promise 靜默 #8 + 框架錯誤 #6）

**問題**：
- **#8 Promise 靜默失敗**：小白最常犯 `async/await` 忘了 `catch`，錯誤完全消失，console 什麼都沒有
- **#6 框架錯誤**：React Error Boundary / Vue errorHandler 吞掉的錯誤不進 console

**修法**：inject.ts 加兩個全域事件監聽，兜住所有漏網之魚。

---

#### §1 inject.ts：加 unhandledrejection 監聽

```ts
window.addEventListener('unhandledrejection', (event) => {
  const reason = event.reason;
  const message = reason instanceof Error
    ? `Unhandled Promise Rejection: ${reason.message}\n${reason.stack || ''}`
    : `Unhandled Promise Rejection: ${String(reason)}`;
  
  collectConsoleLog({
    level: 'error',
    message: message,
    timestamp: Date.now(),
    source: 'unhandledrejection',
  });
  
  // 即時監控也要更新
  updateMonitorCount();
});
```

---

#### §2 inject.ts：加 window.onerror 監聽

```ts
window.addEventListener('error', (event) => {
  // 區分 JS 執行錯誤 vs 資源載入錯誤
  // event.target 是 HTMLElement（img/script/link）→ 資源載入失敗（PM-155 處理）
  // event.target 是 window → JS 執行錯誤
  if (event.target === window || event.target === document) {
    const message = `${event.message || 'Script Error'} at ${event.filename || 'unknown'}:${event.lineno || 0}:${event.colno || 0}`;
    collectConsoleLog({
      level: 'error',
      message: message,
      timestamp: Date.now(),
      source: 'window.onerror',
    });
    updateMonitorCount();
  }
  // 資源載入錯誤在 PM-155 用 capture:true 處理
}, false);  // bubbling phase（JS 錯誤）
```

---

#### §3 避免重複計算

console.error 攔截 + window.onerror + unhandledrejection 可能對同一個錯誤重複記錄。加去重邏輯：

```ts
const recentErrors = new Set<string>();

function dedupeCollect(log: ConsoleLog) {
  const key = `${log.level}:${log.message.slice(0, 100)}`;
  if (recentErrors.has(key)) return; // 已記錄過，跳過
  recentErrors.add(key);
  // 5 秒後清除（允許相同錯誤再次記錄）
  setTimeout(() => recentErrors.delete(key), 5000);
  collectConsoleLog(log);
}
```

所有收集點（console 攔截 / onerror / unhandledrejection）都改呼叫 `dedupeCollect`。

---

#### §4 不改的東西

- server / MCP：不動
- popup / annotate：不動
- console.error / console.warn 攔截（PM-153）：不動，只是多了兩個額外來源

```
cd extension && npm run build
```

2. 測試頁加一個 `undefinedFunction()` → window.onerror 捕捉到
3. 同一個錯誤不重複記錄（去重生效）
