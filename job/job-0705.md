### 🟡 PM-153：Bug 捕捉升級 — console.warn 完整捕捉（CORS #5 + Mixed Content #7）

**問題**：inject.ts 的 console 攔截可能只抓 error 層級，遺漏 warn。CORS 錯誤和 Mixed Content 警告大部分出在 `console.warn`，小白工程師最困惑的兩類問題就這樣漏掉了。

**目標**：確保 inject.ts 攔截 console.warn，且上傳報告 / 即時監控都包含 warn 層級。

---

#### §1 檢查 inject.ts console 攔截

搜尋 inject.ts 中 console 攔截的程式碼，確認是否已攔截 `console.warn`：

```ts
// 應該攔截的層級
const levels = ['error', 'warn'];

levels.forEach(level => {
  const original = console[level];
  console[level] = function(...args: any[]) {
    // 記錄到 BugEzy 收集器
    collectConsoleLog({ level, message: args.map(String).join(' '), timestamp: Date.now() });
    // 呼叫原始方法
    original.apply(console, args);
  };
});
```

如果現在只攔截 `error`，加上 `warn`。

#### §3 報告上傳確認

#### §4 MCP get_console_logs 確認

### 🟡 PM-154：Bug 捕捉升級 — unhandledrejection + window.onerror（Promise 靜默 #8 + 框架錯誤 #6）

**問題**：
- **#8 Promise 靜默失敗**：小白最常犯 `async/await` 忘了 `catch`，錯誤完全消失，console 什麼都沒有

#### §1 inject.ts：加 unhandledrejection 監聽

```ts
window.addEventListener('unhandledrejection', (event) => {
  const reason = event.reason;
  const message = reason instanceof Error
    ? `Unhandled Promise Rejection: ${reason.message}\n${reason.stack || ''}`
    : `Unhandled Promise Rejection: ${String(reason)}`;
  
  collectConsoleLog({
    level: 'error',
    message: message,
    timestamp: Date.now(),
    source: 'unhandledrejection',
  });
  
  // 即時監控也要更新
  updateMonitorCount();
});
```

---

#### §2 inject.ts：加 window.onerror 監聽

```ts
window.addEventListener('error', (event) => {
  // 區分 JS 執行錯誤 vs 資源載入錯誤
  // event.target 是 HTMLElement（img/script/link）→ 資源載入失敗（PM-155 處理）
  // event.target 是 window → JS 執行錯誤
  if (event.target === window || event.target === document) {
    const message = `${event.message || 'Script Error'} at ${event.filename || 'unknown'}:${event.lineno || 0}:${event.colno || 0}`;
    collectConsoleLog({
      level: 'error',
      message: message,
      timestamp: Date.now(),
      source: 'window.onerror',
    });
    updateMonitorCount();
  }
  // 資源載入錯誤在 PM-155 用 capture:true 處理
}, false);  // bubbling phase（JS 錯誤）
```

---

#### §3 避免重複計算

console.error 攔截 + window.onerror + unhandledrejection 可能對同一個錯誤重複記錄。加去重邏輯：

```ts
const recentErrors = new Set<string>();

function dedupeCollect(log: ConsoleLog) {
  const key = `${log.level}:${log.message.slice(0, 100)}`;
  if (recentErrors.has(key)) return; // 已記錄過，跳過
  recentErrors.add(key);
  // 5 秒後清除（允許相同錯誤再次記錄）
  setTimeout(() => recentErrors.delete(key), 5000);
  collectConsoleLog(log);
}
```

所有收集點（console 攔截 / onerror / unhandledrejection）都改呼叫 `dedupeCollect`。

#### §1 inject.ts：Resource 載入失敗捕捉

用 `window.addEventListener('error', ..., true)` 的 **capture phase** 抓資源載入錯誤（img/script/link/video 的 404 等）：

```ts
window.addEventListener('error', (event) => {
  const target = event.target as HTMLElement;
  
  // 只處理資源載入錯誤（非 JS 執行錯誤，JS 在 PM-154 的 bubbling phase 處理）
  if (target === window || target === document) return;
  
  // 取得資源 URL
  const src = (target as HTMLImageElement).src
    || (target as HTMLScriptElement).src
    || (target as HTMLLinkElement).href
    || '';
  
  if (!src) return;
  
  const tagName = target.tagName?.toLowerCase() || 'unknown';
  const message = `Resource load failed: <${tagName}> ${src}`;
  
  dedupeCollect({
    level: 'warn',
    message: message,
    timestamp: Date.now(),
    source: 'resource-error',
  });
  
  updateMonitorCount();
}, true);  // capture phase（資源錯誤不冒泡，必須 capture）
```

---

#### §2 inject.ts：Web Vitals 效能捕捉

用 `PerformanceObserver` 捕捉核心 Web Vitals：

```ts
function collectWebVitals() {
  try {
    // LCP (Largest Contentful Paint) — 最大內容繪製時間
    new PerformanceObserver((list) => {
      const entries = list.getEntries();
      const last = entries[entries.length - 1] as any;
      if (last) {
        collectPerformanceMetric('LCP', Math.round(last.startTime), 'ms');
      }
    }).observe({ type: 'largest-contentful-paint', buffered: true });

    // CLS (Cumulative Layout Shift) — 累計版面位移
    let clsValue = 0;
    new PerformanceObserver((list) => {
      for (const entry of list.getEntries() as any[]) {
        if (!entry.hadRecentInput) clsValue += entry.value;
      }
      collectPerformanceMetric('CLS', Math.round(clsValue * 1000) / 1000, '');
    }).observe({ type: 'layout-shift', buffered: true });

    // FID (First Input Delay) — 首次輸入延遲
    new PerformanceObserver((list) => {
      const entry = list.getEntries()[0] as any;
      if (entry) {
        collectPerformanceMetric('FID', Math.round(entry.processingStart - entry.startTime), 'ms');
      }
    }).observe({ type: 'first-input', buffered: true });
  } catch (e) {
    // 部分瀏覽器不支援某些 observer，靜默失敗
  }
}

function collectPerformanceMetric(name: string, value: number, unit: string) {
  // 判斷是否超標
  const thresholds: Record<string, [number, string]> = {
    'LCP': [2500, '慢（超過 2.5 秒）'],
    'CLS': [0.1, '版面位移過大'],
    'FID': [100, '互動延遲過高'],
  };
  const [threshold, desc] = thresholds[name] || [Infinity, ''];
  const status = value > threshold ? '⚠️ ' + desc : '✅ 良好';

  dedupeCollect({
    level: value > threshold ? 'warn' : 'info',
    message: `Web Vital ${name}: ${value}${unit} ${status}`,
    timestamp: Date.now(),
    source: 'web-vitals',
  });
}

// 頁面載入後執行
collectWebVitals();
```

### 🟡 PM-156：Bug 捕捉升級 — 網路環境快照

**背景**：小白常遇到「我的電腦上好的，客戶那邊就壞」。原因通常是網路環境不同——3G vs WiFi、高延遲、離線等。AI 看不到網路狀態就無法診斷。

**目標**：在錄製/截圖/即時監控上傳報告時，自動附帶網路環境快照。

---

#### §1 inject.ts：收集網路環境

```ts
function getNetworkSnapshot(): Record<string, unknown> {
  const conn = (navigator as any).connection;
  return {
    online: navigator.onLine,
    effectiveType: conn?.effectiveType || 'unknown',  // '4g', '3g', '2g', 'slow-2g'
    rtt: conn?.rtt ?? null,                            // 延遲 ms
    downlink: conn?.downlink ?? null,                  // 頻寬 Mbps
    saveData: conn?.saveData ?? false,                 // 省流量模式
    type: conn?.type || 'unknown',                     // 'wifi', 'cellular', 'ethernet'
  };
}
```

`navigator.connection` 是 Chrome 61+ 支援的 Network Information API，BugEzy 的目標瀏覽器完全沒問題。

---

#### §2 錄製時捕捉

在錄製開始時（startRecording）和結束時（stopRecording）各抓一次網路快照，存入報告 payload：

```ts
const reportPayload = {
  // ...現有欄位
  networkSnapshot: {
    atStart: getNetworkSnapshot(),
    atEnd: getNetworkSnapshot(),
  },
};
```

這樣 AI 能看到「錄製開始時 4G、結束時離線」→ 診斷 API 失敗原因。

---

#### §3 即時監控上傳也帶

PM-124 的「📤 上傳報告讓 AI 分析」也附帶網路快照。透過 postMessage 傳給 content → background：

```ts
window.postMessage({
  type: 'BUGEZY_UPLOAD_MONITOR_REPORT',
  errors: collectedErrors,
  networkSnapshot: getNetworkSnapshot(),
}, '*');
```

---

#### §4 截圖標注也帶

annotate.ts 上傳報告時也附帶 `getNetworkSnapshot()`。annotate 頁面可以直接用 navigator API（不是 MAIN world 限制的 inject.ts）。

---

#### §5 Server：createReport 接收 networkSnapshot

server 的 createReport 把 networkSnapshot 存進 Supabase reports 表。如果 reports 表沒有 networkSnapshot 欄位，加一個 JSONB 欄位：

