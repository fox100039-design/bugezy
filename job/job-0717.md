### 🟡 PM-237：即時字幕體驗優化（interim 文字 + 語音面板可拖曳）

**背景**：FOX 實測日語即時字幕發現兩個體驗問題：

#### Bug 1：底部字幕條沒有即時顯示辨識中的文字

**現象**：底部字幕條一直停在「🟢 聞き取り中...」，直到一整句說完才短暫跳出 `✅ 確認文字`，1.5 秒後又回到「聞き取り中...」。使用者說話時看不到任何即時反饋。

**原因**：`inject.ts` 的 `createRecognition()` 裡 `rec.interimResults = false`，Web Speech API 只送 final 結果，不送暫時（interim）結果。

**修法**：

1. **`rec.interimResults = true`**（改一個字）

2. **`onresult` handler 加入 interim 處理**——目前只處理 `isFinal === true`，需要在 `isFinal === false` 時把暫時文字即時更新到底部字幕條：
   ```
   if (!e.results[i].isFinal) {
     // 暫時文字 → 底部字幕即時顯示（灰色或淺色，表示還在辨識中）
     setCaptionText('🟢 ' + interimText);
   }
   ```
   - interim 文字只更新底部字幕條（`setCaptionText`），**不**推入 `voiceSegments`、**不** flush、**不**加到右上面板
   - 當同一句變成 final 時，既有邏輯接手（推入 segments + 面板 + `✅` 確認）

3. **效果**：使用者說話時，底部字幕條會即時顯示正在辨識的文字（像 YouTube 即時字幕那樣），說完一句後變成 `✅ 確認文字`，1.5 秒後回到「聞き取り中...」等下一句。

#### Bug 2：右上角語音面板（音声記録）改為可拖曳 + 位置記憶

**現象**：語音面板固定在 `top:200px; right:12px`，使用者無法移動。如果面板擋住頁面內容，沒辦法調整。

**需求**：
- 面板可以被使用者拖曳移動（drag）
- 拖曳後記住位置（用 `localStorage` 或直接保持在拖曳後的座標，頁面不重整就不會跑掉）
- 面板用 `position: fixed`（現在已經是），所以捲動頁面不會影響位置——這點已經正確
- 拖曳觸發區：整個 header 列（「🎤 音声記録 ▼」那一條）作為 drag handle，滑鼠按住可拖
- 拖曳時 header cursor 改為 `grab` / `grabbing`

**實作提示**：
- header 加 `cursor: grab`，mousedown 時切 `grabbing`
- mousedown → 記錄 offset → mousemove 更新 panel 的 top/left（改用 left 取代 right，避免計算衝突）→ mouseup 結束
- 邊界限制：不要讓面板拖出視窗外
- `pointer-events` 注意：header 目前是 `pointer-events: none`（PM-31 Bug1），需要**改回 `auto`**（至少 drag handle 區域），但收合按鈕的行為不受影響
- 面板本體的 `pointer-events: none` 也需要改成 `auto`（否則拖不動）

**注意**：
- 不要動到 bottom 字幕條的邏輯
- 面板收合/展開（▼/▶）功能維持不變
- 面板內容區 `bugezy-voice-content` 仍然 `pointer-events: none`（避免誤選文字干擾頁面）

**改動檔案**：只改 `extension/src/inject.ts`

**驗收條件**：
1. 說話時底部字幕條即時顯示辨識中的文字（interim），說完才切成 ✅ final
2. interim 文字不會被推入 voiceSegments 或右上面板（只有 final 才會）
3. 右上語音面板可以拖曳移動，拖到哪就釘在哪
4. 面板捲動頁面不會跑掉（position: fixed）
5. 面板收合/展開正常
6. 拖曳不會超出視窗邊界
7. `npm run build` 無 TS 錯誤

### 🟡 PM-238：修復 🔄 重啟按鈕無限循環（race condition）

**背景**：FOX 實測英文即時字幕，按下底部字幕條的 🔄 重啟按鈕後，系統陷入無限「🟡 Restarting...」循環，無法恢復，且停止錄製變得困難。

**根因**：`forceRestartVoice()` 第 660 行 `recognition.stop()` 會觸發舊實例的 `onend` 事件，而 `onend` 檢查 `voiceActive`（仍然是 `true`）→ 對**舊的** `rec` 執行 `rec.start()` 重啟。隨後 `forceRestartVoice` 又建了**新的**實例並 `start()`。結果兩個 SpeechRecognition 同時搶麥克風，互相觸發 `onend` → 無限循環。

**修法**：在 `forceRestartVoice()` 的 Step 1 中，**先把 `voiceActive` 設為 `false`**，讓 `onend` 看到 `!voiceActive` 直接 return 不自動重啟。等舊實例完全釋放、新實例建好後，再把 `voiceActive` 設回 `true`。

具體改動（`inject.ts` 的 `forceRestartVoice` 函數）：

```typescript
async function forceRestartVoice() {
    blog('手動強制重啟語音');
    if (!voiceActive) return;

    setVoiceStatus('restarting');

    // Step 1：先關 voiceActive，防止 stop() 觸發的 onend 自動重啟舊實例
    voiceActive = false;
    try {
      recognition?.stop();
    } catch { /* 忽略 */ }
    recognition = null;

    // Step 2：getUserMedia 刷新（同既有邏輯）
    try {
      const stream = await navigator.mediaDevices.getUserMedia({ audio: true });
      stream.getTracks().forEach((t) => t.stop());
      blog('getUserMedia 刷新成功');
    } catch (err) {
      blog('getUserMedia 刷新失敗（麥克風可能被封鎖）', err);
      voiceActive = true; // 恢復旗標，讓使用者可再次嘗試
      setVoiceStatus('stopped', it('caption-note-noaccess'));
      return;
    }

    // Step 3：等 500ms（同既有邏輯）
    await new Promise((r) => setTimeout(r, 500));

    // Step 4：恢復 voiceActive，建新實例
    voiceActive = true;
    recognition = createRecognition();
    if (recognition) {
      try {
        recognition.start();
        autoRestartFails = 0;
        setVoiceStatus('listening');
        blog('語音強制重啟成功');
      } catch (err) {
        blog('語音強制重啟失敗', err);
        setVoiceStatus('stopped', it('caption-note-restart-fail'));
      }
    }
  }
```

**改動檔案**：只改 `extension/src/inject.ts` 的 `forceRestartVoice` 函數

**驗收條件**：
1. 即時字幕模式下按 🔄 → 短暫「Restarting...」後回到「🟢 Listening...」，不會無限循環
2. 重啟後語音辨識正常（interim + final 都正常）
3. 停止錄製能正常響應
4. 所有語言通用（en / zh / ja / ko / vi / zh-CN / yue）
5. `npm run build` 無 TS 錯誤

完成後在下方寫 `🔵 DONE-238` + 確認改了什麼。

### 🟡 PM-240：修復韓語即時字幕效能崩潰（interim 節流 + onend 防快速循環）

**背景**：FOX 實測韓語即時字幕，按錄製後整個幾乎崩潰——無法成功錄製、無法停止、無法收音、吃記憶體嚴重。其他語言正常。

**根因（兩個問題疊加）**：

#### 問題 1：interim 事件風暴（DOM 爆炸）
韓語是組合型文字（ㅎ→하→한→한국→한국어），Web Speech API 每個字母組合步驟都觸發 `onresult`，頻率可達每秒數十次。目前每次 interim 都執行 `getElementById + textContent` DOM 更新 → 瀏覽器被 DOM 操作淹沒。

**修法**：對 **interim** 的 `setCaptionText` 加**節流（throttle）**，最多每 150ms 更新一次 DOM。

具體做法——在 `createRecognition` 外面加一個 throttle 機制：
```typescript
let lastInterimUpdate = 0;
const INTERIM_THROTTLE_MS = 150; // 最多每 150ms 更新一次 interim 字幕
```

然後 `onresult` 裡 interim 更新改為：
```typescript
if (!hadFinal && interim.trim()) {
  const now = Date.now();
  if (now - lastInterimUpdate >= INTERIM_THROTTLE_MS) {
    setCaptionText(`🟢 ${interim.trim()}`);
    lastInterimUpdate = now;
  }
}
```

注意：**final 的 `setCaptionText('✅ ...')` 不受節流限制**，確認文字一定即時顯示。

#### 問題 2：onend 無限快速重啟
韓語辨識可能在啟動後極短時間內就觸發 `onend`（<500ms）。目前 `onstart` 無條件把 `autoRestartFails` 歸零（第 597 行），導致失敗計數永遠到不了 3：
```
start → onstart（歸零）→ 瞬間 onend → start → onstart（歸零）→ 瞬間 onend → ∞
```

**修法**：`onstart` 記錄啟動時間，`onend` 判斷——如果本次 session 不到 1 秒就結束，視為「異常短命」，**不歸零失敗計數**：

```typescript
let lastRecognitionStartTime = 0;

rec.onstart = () => {
  if (voiceActive) {
    lastRecognitionStartTime = Date.now();
    // 只有持續超過 1 秒的 session 才歸零失敗計數
    // （短命 session = 疑似異常，保留計數讓 onend 累積到 3 次後停手）
    if (autoRestartFails > 0) {
      // 不在這裡歸零——交給 onend 判斷
    } else {
      setVoiceStatus('listening');
    }
  }
};

rec.onend = () => {
  if (!voiceActive) return;
  const sessionDuration = Date.now() - lastRecognitionStartTime;
  if (sessionDuration > 1000) {
    // 正常 session（超過 1 秒）→ 歸零計數，正常重啟
    autoRestartFails = 0;
  }
  // sessionDuration <= 1000 → 短命 session，不歸零，讓計數累積
  blog(`SpeechRecognition onend（session ${sessionDuration}ms）→ auto restart`);
  setVoiceStatus('restarting');
  try {
    rec.start();
  } catch {
    autoRestartFails++;
    blog(`auto restart 失敗 (第 ${autoRestartFails} 次)`);
    if (autoRestartFails >= 3) {
      setVoiceStatus('stopped', it('caption-note-restart'));
      blog('auto restart 連續失敗 3 次，等待手動重啟');
    }
  }
};
```

等等，上面 `onend` 裡 `rec.start()` 成功但 session 短命的情況——`autoRestartFails` 不歸零，下次 `onend` 再 +1。三次短命 session 後就停手，不再無限循環。但 `onstart` 原本的歸零邏輯要拿掉（改成只在長命 session 的 `onend` 裡歸零）。

更精確的 `onstart`：
```typescript
rec.onstart = () => {
  if (voiceActive) {
    lastRecognitionStartTime = Date.now();
    setVoiceStatus('listening');
    // 不在 onstart 歸零 autoRestartFails——
```

### 🟡 PM-241：越南語 interim 自動升級 final（stale interim 超時機制）

**背景**：FOX 實測越南語即時字幕，底部字幕有即時文字顯示，但右上面板收不到——必須按 🔄 restart 才會把文字送過去。其他語言正常。

**根因**：Chrome Web Speech API 的越南語模型很少主動發出 `isFinal = true`，長時間停在 interim 狀態。只有 `recognition.stop()` 強制結束時 Chrome 才升級為 final。這是語言模型行為差異，不是程式 bug。

**修法**：加入「stale interim 自動升級」——如果 interim 文字**穩定不變超過 3 秒**，自動視為 final，推入 voiceSegments + flush + 加到右上面板。

具體做法（`inject.ts`）：

1. 在 `createRecognition` 外面新增狀態：
```typescript
let interimPromoteTimer: ReturnType<typeof setTimeout> | null = null;
let lastInterimText = '';
```

2. `onresult` 的 interim 區塊末尾（節流之後）加自動升級邏輯：
```typescript
if (!hadFinal && interim.trim()) {
  const trimmed = interim.trim();

  // 既有：節流 150ms 更新字幕（PM-240）
  const now = Date.now();
  if (now - lastInterimUpdate >= INTERIM_THROTTLE_MS) {
    setCaptionText(`🟢 ${trimmed}`);
    lastInterimUpdate = now;
  }

  // 新增：stale interim 自動升級 — 文字穩定 3 秒未變 → 視為 final
  if (trimmed !== lastInterimText) {
    lastInterimText = trimmed;
    if (interimPromoteTimer) clearTimeout(interimPromoteTimer);
    interimPromoteTimer = setTimeout(() => {
      if (lastInterimText && voiceActive) {
        blog('interim 自動升級 final:', lastInterimText.slice(0, 40));
        const seg: VoiceSegment = { text: lastInterimText, timestamp: Date.now(), isFinal: true };
        voiceSegments.push(seg);
        post({ source: BUGEZY_SOURCE, dir: 'to-content', kind: 'FLUSH_VOICE', segment: seg });

        // 右上面板追加
        const vc = document.getElementById('bugezy-voice-content');
        if (vc) {
          vc.textContent += (vc.textContent ? '\n' : '') + lastInterimText;
          const panel = document.getElementById('bugezy-voice-panel');
          if (panel) panel.scrollTop = panel.scrollHeight;
        }

        // 底部字幕確認
        setCaptionText(`✅ ${lastInterimText}`);
        window.setTimeout(() => {
          if (voiceActive) setVoiceStatus('listening');
        }, 1500);

        lastInterimText = '';
      }
    }, 3000); // 3 秒穩定 → 升級
  }
}
```

3. **停止錄製時清除 timer**：在停錄邏輯（`voiceActive = false` 附近）加：
```typescript
if (interimPromoteTimer) { clearTimeout(interimPromoteTimer); interimPromoteTimer = null; }
lastInterimText = '';
```

4. **forceRestartVoice 也清除**：在 Step 1（`voiceActive = false` 之後）加同樣清除。

**對其他語言的影響**：無。英文/中文/日文正常 1-2 秒內就 finalize，3 秒的 timer 永遠不會觸發。只有越南語（和未來可能有類似行為的語言）才會用到。

**改動檔案**：只改 `extension/src/inject.ts`

**驗收條件**：
1. 越南語即時字幕：說完一句話後約 3 秒，右上面板自動收到文字（不需按 restart）
2. 底部字幕先顯示 `🟢 interim`，3 秒穩定後切成 `✅ final`
3. 英文/中文/日文不受影響（正常 finalize，timer 不觸發）
4. 停止錄製時 timer 被清除（不會在停錄後還冒出文字）
5. `npm run build` 無 TS 錯誤

完成後在下方寫 `🔵 DONE-241`。
