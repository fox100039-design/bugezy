#### §3 FAQ 頁面內容（FAQ_PAGE_HTML）

```
🐛 BugEzy 常見問題

━━━━━━━━━━━━━━━━━━━━━━━━━━━

📌 關於產品

Q: BugEzy 是什麼？
A: BugEzy 是一款 Chrome 擴充功能，讓開發者用語音 + 錄製的方式記錄 Bug，AI 透過 MCP 自動讀取報告並提供修復建議。省下 95% 的 debug 溝通時間。

Q: 跟 Jam 有什麼不同？
A: BugEzy 專為亞洲開發者設計，支援中文語音辨識。價格只有 Jam 的 1/5（NT$80 vs $14 USD）。獨家功能：即時監控、30 秒回溯、終端機 CLI、Token 透明度。

Q: 支援哪些 AI 工具？
A: 任何支援 MCP 的 AI 工具都能用，包括 Claude Desktop、Claude Code、Cursor、VS Code + Copilot、Zed、Windsurf、Codex、Replit 等。只需要一行 URL：https://bugezy-api.bugezy-api.workers.dev/mcp

━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔒 關於隱私與安全

Q: BugEzy 會錄到我的密碼嗎？
A: BugEzy 錄製的是 DOM 結構變化，不是螢幕截圖。密碼輸入框（type="password"）的內容會被 rrweb 自動遮蔽，不會錄到實際密碼。

Q: 我的報告誰能看到？
A: 只有你自己。報告預設為私人，只有當你主動分享報告連結時，別人才能看到。

Q: 資料存在哪裡？
A: 報告存在 Cloudflare R2（全球 CDN），使用者資料存在 Supabase（PostgreSQL）。所有傳輸都經過 HTTPS 加密。

━━━━━━━━━━━━━━━━━━━━━━━━━━━

💰 關於方案與付費

Q: 免費版有什麼限制？
A: 免費版每月可錄製 10 次、回溯 5 次、MCP 查詢 20 次。截圖標注和即時監控無限使用。報告保留 7 天。

Q: 付費版多少錢？
A: NT$80/月（約 $3 USD），解鎖全功能無限次使用，報告保留 90 天，加上終端機 CLI、Whisper 精準語音等進階功能。

Q: 如何升級付費版？
A: 在 BugEzy popup 點「升級」按鈕，透過信用卡或 ATM 付款。

Q: 可以取消訂閱嗎？
A: 可以，隨時取消。取消後當月剩餘天數仍可使用付費功能，下個月恢復為免費版。

━━━━━━━━━━━━━━━━━━━━━━━━━━━

🛠 關於技術

Q: 哪些瀏覽器支援？
A: 目前支援 Chrome 和所有 Chromium 瀏覽器（Edge、Brave、Arc 等）。

Q: 會影響網頁效能嗎？
A: 影響極小。BugEzy 只在你主動錄製時才記錄 DOM 變化，即時監控模式只攔截 Console error 和 Network error，不錄 DOM。

Q: MCP 是什麼？
A: Model Context Protocol（模型上下文協議），是 Anthropic 推出的開放標準，讓 AI 工具可以連接外部服務。BugEzy 的 MCP 讓 AI 直接讀取你的 Bug 報告，不需要複製貼上。

Q: Token 是什麼？為什麼 BugEzy 能省 Token？
A: Token 是 AI 處理文字的計量單位，等於你的 AI 使用費用。BugEzy 用結構化文字（而非截圖）傳送報告給 AI，同樣的 Bug 資訊只需要 1/20 的 Token。每次 MCP 查詢都會顯示 Token 估算，讓你看到省了多少。
```

#### §5 手風琴 JS（FAQ 用）

FAQ 的每個問題可點擊展開/收合答案：

```html
<script>
document.querySelectorAll('.faq-q').forEach(q => {
  q.addEventListener('click', () => {
    const a = q.nextElementSibling;
    const isOpen = a.style.maxHeight;
    // 關閉所有
    document.querySelectorAll('.faq-a').forEach(el => { el.style.maxHeight = null; });
    document.querySelectorAll('.faq-q').forEach(el => { el.classList.remove('open'); });
    // 開啟當前
    if (!isOpen) {
      a.style.maxHeight = a.scrollHeight + 'px';
      q.classList.add('open');
    }
  });
});
</script>
```

CSS：
```css
.faq-q { cursor: pointer; padding: 16px; background: #1a1a2e; border-radius: 8px; margin-bottom: 4px; font-weight: 600; display: flex; justify-content: space-between; }
.faq-q::after { content: '▼'; transition: transform 0.2s; }
.faq-q.open::after { transform: rotate(180deg); }
.faq-a { max-height: 0; overflow: hidden; transition: max-height 0.3s; padding: 0 16px; color: #aaa; line-height: 1.8; }
```
