### 🟡 PM-93：Supabase 全 table RLS 鎖死 + 安全根治（Critical）

**問題**：Supabase 安全警告（2026-06-30）— `rls_disabled_in_public`。BugEzy 的 public schema 有 table 沒開 RLS，任何人用 anon key 就能讀寫刪全部資料。

**根本修法**：全部 public table 開啟 RLS + **不加任何 policy** = anon key 完全鎖死。Worker 用 `service_role` key 天生繞過 RLS，不受影響。

**為什麼一勞永逸**：
- RLS 開了沒 policy = deny all，最嚴格的狀態
- 唯一能存取資料的途徑是 Worker 的 service_role
- 以後新增 table 只要記得 `ENABLE ROW LEVEL SECURITY` 就好
- 不需要維護複雜的 policy 規則

---

#### §1 查出所有沒開 RLS 的 table

FOX 在 Supabase SQL Editor 跑：

```sql
SELECT schemaname, tablename, rowsecurity
FROM pg_tables
WHERE schemaname = 'public'
ORDER BY tablename;
```

記錄哪些 table 的 `rowsecurity` 是 `false`。

---

#### §2 對所有 public table 開啟 RLS

根據 §1 結果，對每張 table 執行（已知的 table 列在下面，如有其他的一併加上）：

```sql
-- BugEzy 已知 table
ALTER TABLE reports ENABLE ROW LEVEL SECURITY;
ALTER TABLE mcp_usage ENABLE ROW LEVEL SECURITY;

-- 如果還有其他 table（§1 查出來的），全部加上：
-- ALTER TABLE <table_name> ENABLE ROW LEVEL SECURITY;
```

**不加任何 POLICY**。這樣：
- `service_role`（Worker）：照常讀寫，不受 RLS 限制
- `anon`（公開）：所有 SELECT/INSERT/UPDATE/DELETE 全部 deny

---

#### §3 驗證 Worker 不受影響

開完 RLS 後，用實際操作驗證：

1. **MCP 測試**：呼叫 `list_reports` → 應正常回傳
2. **報告頁**：開一份舊報告 `/report/:id` → 應正常顯示
3. **新錄製**：用 BugEzy 錄一筆 → 上傳 → 報告頁能開 → MCP 能讀
4. **勾選測試**：報告頁勾「高畫質 AI 分析」→ PATCH 應成功

---

#### §4 驗證 anon key 被鎖死

在 Supabase SQL Editor 或用 curl 模擬 anon 存取：

```bash
curl -X GET "https://fpqlclltjreetlyhzlsd.supabase.co/rest/v1/reports?select=*&limit=1" \
  -H "apikey: <ANON_KEY>" \
  -H "Authorization: Bearer <ANON_KEY>"
```

應回傳空陣列 `[]` 或 403，不應看到任何資料。

---

#### §5 防未來遺漏：建立規則

在 ARCHITECTURE.md 加一條鐵律：

> **Supabase 安全鐵律**：所有新 table 必須 `ENABLE ROW LEVEL SECURITY`，不加 policy（deny all）。唯一存取途徑是 Worker 的 `service_role` key。

#### 執行方式

#### 驗收

### 🔴 DONE-93：Supabase RLS 鎖死 + 安全根治 ✅ 程式碼已備妥 + 已部署（⚠ 發現重大前提錯誤，已修正做法）

**🚨 關鍵發現（本 PM 前提與實際 code 相反，直接照跑會弄垮全站）**：
規格假設「Worker 用 service_role key 天生繞過 RLS」——但**實際上 BugEzy 的 Worker 全程用 `SUPABASE_ANON_KEY`**（`createClient` line 93 + `mcp_usage` 兩處 REST fetch）。**三重佐證**：① `Env.SUPABASE_ANON_KEY` + code/wrangler 註解皆寫 anon；② `schema.sql:61` 原有 `ALTER TABLE users DISABLE ROW LEVEL SECURITY`——若 Worker 用 service_role 根本不需要 DISABLE，這證明 Worker 的 key 受 RLS 管轄 = anon。
→ **若現在直接對所有 table 開 RLS(no policy)，Worker(anon) 也會被 deny，全站 Supabase 讀寫全掛（createReport/getReport/MCP/PATCH/googleAuth/ECPay callback/cron 全 500）**。

**做法校正（根治）**：先把 Worker 切到 service_role，再開 RLS。已改 code：
- `Env` 加 `SUPABASE_SERVICE_ROLE_KEY?`；新增 `supaKey(env) = SUPABASE_SERVICE_ROLE_KEY || SUPABASE_ANON_KEY`；`supa()` + 兩處 `mcp_usage` REST fetch 全改走 `supaKey`。
- **安全過渡**：service_role 未設定時**自動退回 anon** → 立即部署也不破壞現況（Version `9a2dc3f6`，cron 仍在）。FOX 設好 service_role secret 後 Worker 自動切換、繞過 RLS。

**我無法直接連 DB**（沒有 service_role key / Management API PAT，那是 FOX 的機密）→ 依規格 fallback **產出完整 SQL 腳本** `server/rls-lockdown.sql`（含：查 RLS 狀態 + 明列 reports/mcp_usage/users + **動態 DO block 對所有 public table 開 RLS**（冪等、涵蓋未列到的）+ 驗證查詢）。`schema.sql` 也同步：`users` 由 DISABLE→ENABLE，末尾加 reports/mcp_usage/users ENABLE + service_role 註記。

**§3 Worker 不受影響驗證**（部署後線上實測，目前走 anon fallback、RLS 尚未開）：MCP `list_reports` → 有報告 ✅；`GET /api/reports/:id` → 200 回報告 ✅；`GET /report/:id` → 200 ✅；`GET /api/usage/monthly`（走 REST fetch 路徑）→ 200 ✅。程式路徑全通，切 service_role 後行為一致（service_role 權限只多不少）。

**§5 ARCHITECTURE.md**：§4 加第 6 條「**Supabase 安全鐵律**」（全 table ENABLE RLS + deny all + 唯一 service_role 存取 + 新 table 必開 RLS + `supaKey` 說明 + 指向 rls-lockdown.sql）；§2/§2a/§6 亦已於 PM-92 更新。

#### §1 offscreen.ts：音量分析 + 定期回報

錄音開始時，從 stream 建 AudioContext + AnalyserNode，每 200ms 回報音量：

```ts
let audioCtx: AudioContext | null = null;
let analyser: AnalyserNode | null = null;
let volumeTimer: ReturnType<typeof setInterval> | null = null;

// startRecording() 裡取得 stream 後加：
audioCtx = new AudioContext();
const source = audioCtx.createMediaStreamSource(stream);
analyser = audioCtx.createAnalyser();
analyser.fftSize = 256;
source.connect(analyser);

const dataArray = new Uint8Array(analyser.frequencyBinCount);
volumeTimer = setInterval(() => {
  if (!analyser) return;
  analyser.getByteFrequencyData(dataArray);
  const avg = dataArray.reduce((a, b) => a + b, 0) / dataArray.length;
  const level = Math.min(avg / 128, 1);
  chrome.runtime.sendMessage({ type: 'MIC_VOLUME', level });
}, 200);
```

stopRecording() 清理：
```ts
if (volumeTimer) { clearInterval(volumeTimer); volumeTimer = null; }
if (audioCtx) { audioCtx.close(); audioCtx = null; analyser = null; }
```

function isDarkBackground(): boolean {
  const bg = window.getComputedStyle(document.body).backgroundColor;
  const match = bg.match(/\d+/g);

}

```ts
function applyOrangePulse(bar: HTMLElement) {
  if (!document.getElementById('bugezy-orange-pulse-style')) {
    const style = document.createElement('style');
    style.id = 'bugezy-orange-pulse-style';
    style.textContent = `
      @keyframes bugezy-orange-pulse {
        0%, 100% { box-shadow: 0 0 8px rgba(255,140,0,0.3); border-color: #ff8c00; }
        50% { box-shadow: 0 0 30px rgba(255,140,0,1), 0 0 60px rgba(255,140,0,0.5); border-color: #ffaa00; }
      }
    `;
    document.head.appendChild(style);
  }
  bar.style.border = '2px solid #ff8c00';
  bar.style.boxShadow = '0 0 20px rgba(255,140,0,0.8)';
  bar.style.animation = 'bugezy-orange-pulse 0.7s ease-in-out 10';

  setTimeout(() => {
    bar.style.animation = 'none';
    bar.style.borderColor = '#444';
    bar.style.boxShadow = '0 2px 8px rgba(124,58,237,0.15)';
    bar.style.transition = 'border-color 0.5s, box-shadow 0.5s';
    document.getElementById('bugezy-orange-pulse-style')?.remove();
  }, 7000);
}
```

### 🟡 PM-105：修復錄製中開麥克風觸發授權頁導致錄製卡死

**問題**：錄製中 → popup 開 mic toggle → 彈 mic-permission.html → 搶焦點 → 錄製卡死、停止失效。

**修法**：錄製中 mic toggle 只存偏好，不開授權頁，下次錄製才生效。

---

#### §1 popup.ts：toggle change 時判斷錄製狀態

mic toggle 的 change handler 加一層判斷：

```ts
if (micToggle.checked && !store[MIC_PERMISSION_KEY]) {
  // 未授權：先問 background 是否在錄製
  const state = await chrome.runtime.sendMessage({ type: 'GET_RECORDING_STATE' });
  if (state?.recording) {
    // 錄製中：存偏好但不開授權頁
    await chrome.storage.local.set({ [MIC_KEY]: true });
    updateMicUI();
    return; // 下次錄製時才會觸發授權
  }
  // 未錄製：正常開授權頁
  micToggle.checked = false;
  updateMicUI();
  await chrome.runtime.sendMessage({ type: 'REQUEST_MIC_PERMISSION' });
  return;
}
```

---

#### §2 background.ts：新增 GET_RECORDING_STATE

```ts
case 'GET_RECORDING_STATE': {
  sendResponse({ recording: recordingTabId !== null });
  break;
}
```

用 `recordingTabId`（PM-97 已建立）判斷是否在錄製中。
