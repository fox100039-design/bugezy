### 🔵 DONE-48：測試專頁（Test Harness）✅ 已部署 + 線上驗證

**改動**：只改 `server/src/index.ts`（extension/web/MCP/schema 皆不動，與規格一致）。

**新增路由**（放在 `/mcp` 之後、`/api/*` try 區塊之前）：
- `GET /test` → `TEST_PAGE_1`（完整測試頁：Console / Network / DOM 變化 / 截圖區 / 跨頁連結 / 輸入框 + 內嵌 script）
- `GET /test/page2` → 第二頁（console.error/warn 鈕 + 回頁1/前往頁3 連結）
- `GET /test/page3` → 第三頁（回頁1 連結 + 20 段長內容測捲動截圖）
- `GET /test/api/:status` → 回傳指定 HTTP status（夾 100–599，否則 200），含 CORS

**實作重點**：
- 抽 `TEST_STYLE`（CSS 單一來源）+ `testShell(pageId, inner)` 骨架函式，page1 與 page2/3 共用樣式。
- page1 內嵌 `<script>` 的 JS 字串換行用 `\\n`（TS 模板字面值 → 送到瀏覽器為 `\n` 兩字元，瀏覽器端才是真正換行；直接寫 `\n` 會在模板字面值裡變成實際換行導致瀏覽器 JS 字串未閉合）。
- page3 長內容用 `Array.from({length:20})` 產生，`${i+1}` 由 TS 端展開、不外洩到 HTML。
- 加 `html()` helper 回 `text/html; charset=utf-8`。

**tsc**：`npx tsc --noEmit` ✅。
**部署**：`npx wrangler deploy` ✅（Version `b1e4803c`）。

**線上實測（curl）全綠**：
- `/test` → HTTP 200 / `text/html`，含「📍 測試頁 1」+ script + `/test/page2` 連結。
