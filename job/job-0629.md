### 🔵 DONE-75b：修 popup 付費用戶升級提示未隱藏 ✅ build 通過

**根因（CSS 特異度/順序 bug，非 JS）**：`popup.html` 裡 `.hidden { display:none }`（line 78）定義在 `.upgrade-hint { display:flex }`、`.manage-sub { display:flex }` **之前**。三者都是單一 class 選擇器（特異度相同），CSS 同特異度時**後定義者勝** → 帶 `class="upgrade-hint hidden"` 的元素仍套到 `display:flex`、**`hidden` 失效**。所以 PM-75 的 `loadPlan` 雖正確 `upgradeHint.classList.add('hidden')`，畫面上升級提示仍蓋掉、付費用戶同時看到「✨ 付費版 + 取消訂閱」和「升級付費版」。

**修法（CSS 層統一收斂，一處解決）**：`.hidden { display: none !important; }`——讓 `.hidden` 永遠勝過任何 `display:flex/block`，不受定義順序/特異度影響。一次修好 `upgradeHint`、`manageSubscription` 及所有用 `.hidden` 的元素（loginView/mainView/idle·recording·doneView 本來就靠 `.hidden`，加 `!important` 不影響其正確行為）。

**JS 端確認已收斂**：`upgradeHint` 顯示/隱藏只在兩處——① `loadPlan()`（plan 狀態權威來源：paid/cancelled `add('hidden')`、free `remove('hidden')`）② `startBtn` 的 `limitReached` 路徑（免費版用滿額度的執行期觸發；伺服器只對非付費回 limit_reached，付費不會誤觸）。兩者語意不重疊，無需再合併。

**Build**：`npm run build` ✅ + `tsc --noEmit` ✅；確認 `dist/popup.html` 含 `display: none !important`。

**🟡 提醒 FOX**：① 這是真正的 root cause（PM-75 改 JS 是對的，但被 CSS 蓋掉所以沒效果）。② **本 PM 只 build、未重打包 zip**（依規格）——重新上傳 Chrome Web Store 前請先重打包 `bugezy-v0.1.0.zip`（要的話我可代勞），目前磁碟上的 zip 仍是 PM-76 版（不含此 CSS 修正）。③ 實機驗收：付費/已取消用戶開 popup 應只見「✨ 付費版/已取消」卡片，**不再**有升級提示。
