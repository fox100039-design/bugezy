### 🟡 PM-75：修 popup 付費用戶仍顯示升級提示

**問題**：已付費的用戶 popup 還是顯示「升級付費版 NT$80/月」。

**修法**：popup.ts 裡查 plan 後，根據狀態控制 UI：

```ts
if (plan.plan === 'paid') {
  // 隱藏升級提示
  upgradeHint.classList.add('hidden');
  // 按鈕不顯示剩餘次數
  recordBtn.textContent = '🎬 錄製 ✨';
  // 顯示管理訂閱
  manageSubscription.classList.remove('hidden');
} else if (plan.plan === 'cancelled') {
  // 顯示到期日，隱藏升級提示
  upgradeHint.classList.add('hidden');
  // 顯示到期日提示
} else {
  // 免費版：顯示剩餘次數 + 升級提示
  upgradeHint.classList.remove('hidden');
}
```

確保：
1. paid 用戶：不顯示升級提示，按鈕顯示 ✨，顯示「管理訂閱」
2. cancelled 用戶：不顯示升級提示，顯示到期日
3. free 用戶：顯示剩餘次數 + 升級提示

### 🔵 DONE-75：修 popup 付費用戶仍顯示升級提示 ✅ build 通過 + zip 已重打包

**改動**：只改 `extension/src/popup.ts` 的 `loadPlan()`（server/html/其他不動）。

**重點**：把 plan 狀態判斷由原本「看 `plan.limits` 是否為 null」改成**直接以 `plan.plan` 為準**（source of truth），三態明確分流：
- **paid** → `setRecordDesc('✨ 無限次')`、`upgradeHint` 隱藏、`subStatus='✨ 付費版'`、顯示取消連結 + `manageSubscription` 顯示。
- **cancelled** → `✨ 無限次`、`upgradeHint` 隱藏、`subStatus='已取消，可用到 YYYY/MM/DD'`、隱藏取消連結、`manageSubscription` 顯示。
- **free（含未知狀態 fallback）** → `manageSubscription` 隱藏、有 `limits` 才顯示剩餘次數（`剩 N 次`/`已用完（升級解鎖）`）、`upgradeHint` 顯示。

> 為何改：原本靠 `plan.limits` 真值判斷是「間接」推斷付費，若伺服器回傳形狀變動易誤判；改以 `plan.plan` 直接判斷更穩。免費分支用 `plan.limits?.recording` 防呆（沒 limits 也不會炸）。
