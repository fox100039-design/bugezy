### 🟡 PM-82：報告頁截圖可視化勾選 + MCP 連動

**背景**：PM-57 已完成 MCP `get_screenshots` 的 `include_images` 分層控制，但目前只有 AI 自己決定是否帶 `include_images: true`。使用者無法主動告訴 AI「我這張截圖有視覺問題，請看圖」。

**目標**：在報告頁 `/report/:id` 新增一個勾選開關，讓使用者自己決定是否允許 AI 讀取截圖圖片。勾了 → MCP `get_screenshots` 自動回傳圖片，不用 AI 猜。

---

#### §1 Supabase 新增欄位

`reports` 表加一個欄位：

```sql
ALTER TABLE reports ADD COLUMN allow_screenshot_images BOOLEAN DEFAULT false;
```

`false` = AI 只拿 metadata（省 token）
`true` = AI 自動拿到圖片（視覺 Bug 用）

#### §2 報告頁 UI（server-rendered HTML）

#### §3 前端 JS：勾選即時儲存

```js

const hint = document.getElementById('toggle-hint');
const token = document.getElementById('toggle-token');
```

#### §5 MCP `get_screenshots` 改動

目前邏輯：依 `include_images` 參數決定是否回傳圖片。

新邏輯：**兩層判斷**

```ts
// 1. 讀報告設定
const { data } = await supabase()
  .from('reports')
  .select('screenshots_r2_key, screenshot_count, allow_screenshot_images')
  .eq('report_id', report_id)
  .single();

// 2. 決定是否回傳圖片
//    使用者勾了 OR AI 明確要求 → 回傳圖片
const shouldIncludeImages = data.allow_screenshot_images || include_images;
```

這樣的好處：
- 使用者勾了 → AI 不用帶參數就自動看到圖
- 使用者沒勾 → AI 仍然可以用 `include_images: true` 主動要求看圖
- 兩邊都有控制權，不衝突

#### §3 handleTranscribe 實作

```ts
async function handleTranscribe(request: Request, env: Env): Promise<Response> {
  // 1. 讀取音訊（multipart form-data 或 raw binary）
  const contentType = request.headers.get('content-type') || '';
  let audioBlob: Blob;

  if (contentType.includes('multipart/form-data')) {
    const formData = await request.formData();
    const file = formData.get('audio');
    if (!file || !(file instanceof File)) {
      return json({ error: '缺少 audio 欄位' }, 400);
    }
    audioBlob = file;
  } else {
    // raw binary（extension 直接送 blob）
    audioBlob = await request.blob();
  }

  // 2. 檢查大小（上限 25MB，Groq 免費版限制）
  if (audioBlob.size > 25 * 1024 * 1024) {
    return json({ error: '音訊超過 25MB 上限' }, 400);
  }
  if (audioBlob.size < 100) {
    return json({ error: '音訊太短' }, 400);
  }

  // 3. 呼叫 Groq Whisper API
  const groqForm = new FormData();
  groqForm.append('file', audioBlob, 'audio.webm');
  groqForm.append('model', 'whisper-large-v3-turbo');
  groqForm.append('language', 'zh');          // 預設中文
  groqForm.append('response_format', 'verbose_json');

  try {
    const groqRes = await fetch('https://api.groq.com/openai/v1/audio/transcriptions', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${env.GROQ_API_KEY}`,
      },
      body: groqForm,
    });

    if (!groqRes.ok) {
      const errText = await groqRes.text();
      console.error('Groq API error:', groqRes.status, errText);
      return json({ error: 'Groq 轉錄失敗', detail: errText }, 502);
    }

    const result = await groqRes.json() as {
      text: string;
      segments?: Array<{ start: number; end: number; text: string }>;
      language?: string;
      duration?: number;
    };

    return json({
      ok: true,
      text: result.text,
      segments: result.segments || [],
      language: result.language || 'zh',
      duration: result.duration || 0,
    });

  } catch (err) {
    console.error('Groq fetch error:', err);
    return json({ error: '語音轉錄服務暫時不可用' }, 503);
  }
}
```

```ts
// 監聽 background 指令
chrome.runtime.onMessage.addListener((msg, _sender, sendResponse) => {
  if (msg.type === 'OFFSCREEN_START_MIC') {
    startRecording();
    sendResponse({ ok: true });
  } else if (msg.type === 'OFFSCREEN_STOP_MIC') {
    stopRecording().then(sendResponse);
    return true; // async
  }
});

let mediaRecorder: MediaRecorder | null = null;
let chunks: Blob[] = [];

async function startRecording() {
  try {
    const stream = await navigator.mediaDevices.getUserMedia({
      audio: {
        channelCount: 1,
        sampleRate: 16000,  // Whisper 最佳
        echoCancellation: true,
        noiseSuppression: true,
      }
    });
    chunks = [];
    mediaRecorder = new MediaRecorder(stream, {
      mimeType: 'audio/webm;codecs=opus',
    });
    mediaRecorder.ondataavailable = (e) => {
      if (e.data.size > 0) chunks.push(e.data);
    };
    mediaRecorder.start(1000); // 每秒一個 chunk
    console.log('[BugEzy offscreen] 錄音開始');
  } catch (err) {
    console.error('[BugEzy offscreen] getUserMedia 失敗:', err);
  }
}

async function stopRecording(): Promise<{ audioBlob?: string; error?: string }> {
  return new Promise((resolve) => {
    if (!mediaRecorder || mediaRecorder.state === 'inactive') {
      resolve({ error: '未在錄音中' });
      return;
    }
    mediaRecorder.onstop = async () => {
      // 合併 chunks → blob → base64
      const blob = new Blob(chunks, { type: 'audio/webm;codecs=opus' });
      chunks = [];
      // 停止所有音軌
      mediaRecorder?.stream.getTracks().forEach(t => t.stop());
      mediaRecorder = null;

      if (blob.size < 100) {
        resolve({ error: '錄音太短' });
        return;
      }

      // 轉 base64 傳給 background
      const reader = new FileReader();
      reader.onloadend = () => {
        resolve({ audioBlob: reader.result as string });
      };
      reader.readAsDataURL(blob);
    };
    mediaRecorder.stop();
  });
}
```
