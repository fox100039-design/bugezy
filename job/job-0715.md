### 🟡 PM-232：簡體中文支援（zh-CN）— Extension + Server

**背景**：BugEzy 目前支援繁體中文（zh-TW）、粵語（yue）、英文（en）。簡體中文是最大的潛在市場（中國 + 東南亞華人），且改動最小（繁體轉簡體 + 少量用語調整）。

**需要改的地方：**

#### ① Extension 端

**popup 語言選擇 dropdown**：
- 新增選項 `🇨🇳 简体中文`，value = `zh-CN`
- 放在 `🇹🇼 繁體中文` 下方、`🇭🇰 粵語` 上方
- 不是 disabled（直接可用）

**types.ts**：
- `SupportedLang` 加入 `'zh-CN'`
- `SPEECH_LANG_MAP` 加入 `'zh-CN': 'zh-CN'`（Web Speech API 簡體中文）

**i18n.ts**：
- `getUILang()` 判斷 `zh-CN` → 回傳新的 UI 語系 `'zh-CN'`（不是 `'zh'`）
- 所有 `T('xxx')` 的翻譯字串新增 `'zh-CN'` 版本
- 翻譯原則：把繁體中文版的文字轉簡體 + 調整用語（擴充功能→扩展程序、截圖→截图、錄製→录制、報告→报告、語音→语音、設定→设置）

**主要需翻譯的 UI 文字（繁→簡對照）：**
- 語音 Bug 回報 → 语音 Bug 回报
- 錄製 → 录制
- 回溯 30s → 回溯 30s（不變）
- 截圖標注 → 截图标注
- 我的報告 → 我的报告
- 付費版會員 → 付费版会员
- 進階設定 → 高级设置
- 語音語言 → 语音语言
- 即時字幕 → 实时字幕
- 精準轉錄 → 精准转录
- 聽取中... → 听取中...
- 停止錄製 → 停止录制
- 編輯報告頁所有文字 → 同步翻譯簡體
- 麥克風授權頁 → 同步翻譯簡體

**edit-report.ts**：
- `speechToSrLang` 加 `zh-CN` → `'zh-CN'`
- AI 校正/精簡的 language 判斷：`zh-CN` 走中文 prompt（跟 `zh` 一樣用繁體 prompt 就好，Whisper 會輸出簡體）

**inject.ts**：
- 即時字幕浮動條 i18n 加 `zh-CN` → 「听取中...」

**mic-permission.html**：
- i18n 加 `zh-CN` 簡體版文字

#### ② Server 端

**Whisper prompt**：
- `zh-CN` 用不同的 prompt：`'以下是简体中文的语音转录内容。'`（引導輸出簡體）
- 不能用繁體 prompt，否則簡體用戶的語音會輸出繁體字

**AI 校正/精簡 prompt**：
- `zh-CN` 用簡體中文 prompt：`'你是简体中文语音转文字的校对专家...'`

**官網 getLang / detectLang**：
- `Accept-Language` 包含 `zh-CN` 或 `zh-Hans` → 回傳 `'zh-CN'`
- `Accept-Language` 包含 `zh-TW` 或 `zh-Hant` 或 `zh-HK` → 回傳 `'zh'`（繁體，現有行為）
- 只有 `zh`（無地區碼）→ 保持現有邏輯回 `'zh'`

**官網所有頁面中文版**：
- 新增 `zh-CN` 版本（簡體中文），或至少首頁 + features 頁
- 可以用 `?lang=zh-CN` 手動切換

**注意事項**：
- 繁體轉簡體不是單純字體轉換，有用語差異（設定→设置、程式→程序、擴充→扩展）
- 簡體中文在 Chrome Web Store 是獨立的 locale，可以增加搜尋能見度

**驗收條件**：
1. popup 能選「简体中文」，UI 顯示簡體
2. 即時字幕 + 精準轉錄用簡體中文辨識
3. Whisper 輸出簡體字（不是繁體）
4. edit-report 頁面 UI 簡體
5. 官網 `?lang=zh-CN` 顯示簡體版

完成後在下方寫 `🔵 DONE-232` + 改了哪些檔案。
