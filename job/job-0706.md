#### §2 Server bumpUsage：每月自動重置

bumpUsage 開頭檢查 usage_reset_at，超過 30 天就歸零：

```ts
const resetAt = new Date(u.usage_reset_at || 0);
const now = new Date();
const daysSinceReset = (now.getTime() - resetAt.getTime()) / (1000 * 60 * 60 * 24);

if (daysSinceReset >= 30) {
  await supa(env).from('users').update({
    recording_count: 0, rewind_count: 0, mcp_count: 0,
    usage_reset_at: now.toISOString(),
  }).eq('user_id', userId);
  u.recording_count = 0;
  u.rewind_count = 0;
  u.mcp_count = 0;
}
```

bumpUsage 的 select 加 `usage_reset_at`。

#### §4 Extension background.ts：回溯加用量檢查

新增 checkRewindUsage()，仿照 checkRecordingUsage()：
```ts
async function checkRewindUsage(): Promise<string | null> {
  const headers = await getAuthHeaders();
  if (!headers.Authorization) return null;
  try {
    const res = await fetch(`${API_BASE}/api/user/usage`, {
      method: 'POST', headers,
      body: JSON.stringify({ type: 'rewind' }),
    });
    if (res.status === 403) {
      const err = await res.json();
      if (err.error === 'limit_reached') return err.message || '回溯額度已用完';
    }
    return null;
  } catch { return null; }
}
```

在 startRewind() 最前面呼叫，達上限不進入回溯。

#### §3 Server：POST /api/feedback

```ts
if (method === 'POST' && path === '/api/feedback') {
  const body = await request.json() as { email?: string; category: string; message: string };
  
  if (!body.message || !body.category) {
    return json({ error: '請填寫問題描述' }, 400);
  }

  // 訊息長度限制
  if (body.message.length > 5000) {
    return json({ error: '描述過長，請控制在 5000 字內' }, 400);
  }

  const country = (request as any).cf?.country || 'UNKNOWN';

  await supa(env).from('feedback').insert({
    email: body.email?.slice(0, 200) || null,
    category: body.category,
    message: body.message.slice(0, 5000),
    user_agent: request.headers.get('User-Agent')?.slice(0, 500) || '',
    country: country,
  });

  return json({ ok: true });
}
```

不需要登入就能回報（降低門檻）。有 rate limiting 保護（Cloudflare /api/ 10req/10s）。
