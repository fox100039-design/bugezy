### 🟡 PM-128：P0 修復 — Server 端 session token 認證（取代假 base64）

**問題**：`getUserIdFromHeader` 只做 base64 decode，無簽章驗證。任何人可偽造 Authorization header 冒充其他使用者。金流相關端點（取消訂閱、買日票）都受影響。

**修法**：改為 session token 驗證——登入時 server 產生隨機 token 存 DB，後續 API 呼叫用 token 查 DB 取得 user_id。

---

#### §1 Supabase 新增 sessions 表

FOX 需在 SQL Editor 手動跑：

```sql
CREATE TABLE IF NOT EXISTS sessions (
  session_token TEXT PRIMARY KEY,
  user_id TEXT NOT NULL REFERENCES users(user_id),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  expires_at TIMESTAMPTZ DEFAULT (NOW() + INTERVAL '90 days')
);

-- RLS 鎖死（只有 service_role 能存取）
ALTER TABLE sessions ENABLE ROW LEVEL SECURITY;
```

#### §2 Server：新增 POST /api/auth/session

擴充登入 Google OAuth 成功後，呼叫此端點換取 session token：

```ts
async function createSession(request: Request, env: Env): Promise<Response> {
  const body = await request.json() as { user_id: string; email: string; name?: string };
  if (!body.user_id || !body.email) {
    return json({ error: 'missing user_id or email' }, 400);
  }

  // 驗證 user 存在（或自動建立）
  const { data: user } = await supa(env)
    .from('users')
    .select('user_id')
    .eq('user_id', body.user_id)
    .maybeSingle();

  if (!user) {
    // 首次登入：建立 user
    await supa(env).from('users').insert({
      user_id: body.user_id,
      email: body.email,
      name: body.name || '',
      plan: 'free',
    });
  }

  // 產生 session token（crypto 隨機，不可猜測）
  const sessionToken = crypto.randomUUID() + '-' + crypto.randomUUID();

  // 存入 sessions 表
  await supa(env).from('sessions').insert({
    session_token: sessionToken,
    user_id: body.user_id,
    expires_at: new Date(Date.now() + 90 * 24 * 60 * 60 * 1000).toISOString(),
  });

  return json({ session_token: sessionToken });
}
```

#### §3 Server：改寫 getUserIdFromHeader → verifySession

```ts
async function verifySession(request: Request, env: Env): Promise<string | null> {
  const auth = request.headers.get('Authorization');
  if (!auth) return null;
  const token = auth.replace('Bearer ', '').trim();
  if (!token || token.length < 10) return null;

  // 查 DB 驗證
  const { data } = await supa(env)
    .from('sessions')
    .select('user_id, expires_at')
    .eq('session_token', token)
    .maybeSingle();

  if (!data) return null;

  // 檢查是否過期
  if (new Date(data.expires_at) <= new Date()) {
    // 過期：刪除 session
    await supa(env).from('sessions').delete().eq('session_token', token);
    return null;
  }

  return data.user_id;
}
```

#### §4 全站替換 getUserIdFromHeader → verifySession

#### §7 保留舊的 getUserIdFromHeader 做過渡

短期內舊版擴充還在用舊的 base64 header。可以做 fallback：

```ts
async function getAuthUserId(request: Request, env: Env): Promise<string | null> {

  const sessionUserId = await verifySession(request, env);

  // 過渡期：舊版 base64（等全部使用者更新後移除）

}
```

#### §2 統一的 API 呼叫 header

建一個 helper 函式，所有 API 呼叫統一用：

```ts
async function getAuthHeaders(): Promise<Record<string, string>> {
  const store = await chrome.storage.local.get('bugezy:session-token');
  const token = store['bugezy:session-token'];
  return {
    'Content-Type': 'application/json',
    ...(token ? { 'Authorization': `Bearer ${token}` } : {}),
  };
}
```
