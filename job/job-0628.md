#### §1 popup 更新通知

版本號從 manifest.json 讀取。用 chrome.storage.local 記錄上次顯示的版本，如果不同就顯示更新提示。

**popup.ts 加：**

```ts
// 版本更新通知
const manifest = chrome.runtime.getManifest();
const currentVersion = manifest.version;
const stored = await chrome.storage.local.get('bugezy:lastVersion');
const lastVersion = stored['bugezy:lastVersion'];

if (lastVersion && lastVersion !== currentVersion) {
  showUpdateNotice(currentVersion);
}
await chrome.storage.local.set({ 'bugezy:lastVersion': currentVersion });
```

**showUpdateNotice 函式：**

```ts
function showUpdateNotice(version: string) {
  const notice = document.createElement('div');
  notice.className = 'update-notice';
  notice.innerHTML = `
    <div class="update-title">🎉 BugEzy 更新到 v${version}</div>
    <div class="update-body">感謝使用 BugEzy！此版本改善了穩定度和使用體驗。</div>
    <button class="update-dismiss" id="dismissUpdate">知道了</button>
  `;
  document.body.prepend(notice);
  
  document.getElementById('dismissUpdate')?.addEventListener('click', () => {
    notice.remove();
  });
}
```

#### §2 打包 Chrome Web Store zip

Chrome Web Store 需要一個 zip 檔包含擴充功能的所有檔案。

**打包規則：**
- 只包含 extension/ 目錄下的必要檔案
- 包含：manifest.json, popup.html, popup.js, background.js, content.js, inject.js, annotate.html, annotate.js, edit-report.html, edit-report.js, offscreen.html, offscreen.js, icons/, styles/ 等 dist/build 輸出的檔案
- 不包含：src/, node_modules/, tsconfig.json, package.json, esbuild 設定等開發檔案

**步驟：**

```bash
cd C:\dev\bugezy\extension
npm run build

# 建立乾淨的打包目錄
mkdir C:\dev\bugezy\dist-zip
# 複製必要檔案到 dist-zip
# 然後壓縮成 bugezy-v0.1.0.zip
```

Claude Code 請：
1. 先 `npm run build` 確保最新
2. 讀 manifest.json 確認版本號
3. 把需要的檔案複製到 `C:\dev\bugezy\dist-zip\` 目錄
4. 用 PowerShell `Compress-Archive` 壓縮成 `C:\dev\bugezy\bugezy-v{version}.zip`
5. 列出 zip 內容確認
