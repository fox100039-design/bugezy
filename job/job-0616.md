```ts
let events: any[] = [];
```

**B. Console 攔截（只抓 warn + error）**

```ts
const consoleLogs: Array<{ level: string; message: string; timestamp: number }> = [];
const originalWarn = console.warn;
const originalError = console.error;

console.warn = (...args) => {
  consoleLogs.push({ level: 'warn', message: args.join(' '), timestamp: Date.now() });
  originalWarn.apply(console, args);
};
console.error = (...args) => {
  consoleLogs.push({ level: 'error', message: args.join(' '), timestamp: Date.now() });
  originalError.apply(console, args);
};
```

**C. Network 攔截（只抓 4xx/5xx）**

用 `fetch` 和 `XMLHttpRequest` 的 monkey patch：

```ts
const networkErrors: Array<{
  method: string; url: string; status: number;
  requestBody?: string; responseBody?: string;
  timestamp: number; duration: number;
}> = [];

const originalFetch = window.fetch;
window.fetch = async (...args) => {
  const start = Date.now();
  const response = await originalFetch(...args);
  if (response.status >= 400) {
    const body = await response.clone().text().catch(() => '');
    networkErrors.push({
      method: (args[1]?.method || 'GET'),
      url: typeof args[0] === 'string' ? args[0] : args[0].url,
      status: response.status,
      responseBody: body.slice(0, 2000),
      timestamp: start,
      duration: Date.now() - start,
    });
  }
  return response;
};
// XMLHttpRequest 同理
```

### 🟡 PM-07：修語音 — popup 先取麥克風權限 + offscreen 握手

**問題**：PM-06 語音完全沒收到（voiceTranscript 空陣列）。Jam 錄影確認 Chrome 沒有跳麥克風授權提示。

**根因**：offscreen document 是隱藏頁面，Chrome 不為它彈權限提示。`SpeechRecognition.start()` 靜默失敗（沒有拋錯到頁面 console，因為 offscreen console 在 service worker 那邊）。

**修法兩件事**：

---

#### Fix 1：Popup 先要麥克風權限

**`src/popup.ts`** — 在 startBtn click handler 裡，**送 START_RECORDING 之前**，先搶麥克風：

```ts
startBtn.addEventListener('click', async () => {
  startBtn.disabled = true;
  try {
    // ── 先確保麥克風權限（popup 是可見視窗，會彈授權）──
    try {
      const stream = await navigator.mediaDevices.getUserMedia({ audio: true });
      // 拿到就立刻關，我們不需要這個 stream，只要權限
      stream.getTracks().forEach(t => t.stop());
    } catch (micErr) {
      // 使用者拒絕或無麥克風 → 照錄，只是沒語音
      console.warn('[BugEzy popup] 麥克風取得失敗，語音將不可用', micErr);
    }
    // ── 原有邏輯 ──
    render(await send<StateResponse>('START_RECORDING'));
    // ...
  } catch (err) { ... }
});
```

**重點**：
- `getUserMedia` 會觸發 Chrome 的授權提示（popup 是可見視窗，一定會彈）。
- 拿到 stream 後**立刻 stop**（我們不要這個 stream，只是讓 Chrome 記住「此擴充有麥克風權限」）。
- 同一個 extension origin（`chrome-extension://<id>`）下，popup 授權後 offscreen 也能用。
- 如果使用者拒絕 → `catch` 吃掉，照常錄製（只是語音為空），**不中斷 DOM/console/network 錄製**。
- 第二次以後 Chrome 記住了，不會再彈。

---

#### Fix 2：Offscreen 握手（解決競態）

**問題**：background 在 `createDocument()` 完成後立刻送 `VOICE_START`，但 offscreen.ts 可能還沒載入完、listener 還沒註冊，訊息就丟了。

**`src/offscreen.ts`** — 載入最後加：


**`src/types.ts`** — ControlMessage 加：
```ts

  const tab = await getActiveTab();
  if (!tab?.id) throw new Error('找不到 active tab');

  // 1. 建 offscreen（如果不存在）
  await ensureOffscreen();
  // 2. 清空語音暫存
  await chrome.storage.local.set({ [VOICE_KEY]: [] });

  // 3. 等 offscreen 回報 READY，再送 VOICE_START
  //    （用 Promise 包 onMessage 一次性監聽，3 秒 timeout）
  const voiceReady = new Promise<void>((resolve) => {
    const timer = setTimeout(() => {
      blog('⚠ offscreen VOICE_READY 超時（3s），語音可能不可用');
      resolve(); // 超時也繼續，不阻擋錄製
    }, 3000);
    const handler = (msg: { type: string }) => {
      if (msg.type === 'VOICE_READY') {
        clearTimeout(timer);
        chrome.runtime.onMessage.removeListener(handler);
        resolve();
      }
    };
    chrome.runtime.onMessage.addListener(handler);
  });

  await voiceReady;
  // 現在 offscreen 一定準備好了
  chrome.runtime.sendMessage({ type: 'VOICE_START' });

  // 4. 送 START 給 content tab（原有邏輯）
  await chrome.tabs.sendMessage(tab.id, { type: 'START_RECORDING' } satisfies ControlMessage);
  const s = await setState({
    recording: true,
    startedAt: Date.now(),
    tabId: tab.id,
    summary: null,
  });
  setBadgeRecording();
  return toResponse(s);
}
```

```ts
            blog('voice segment:', text.slice(0, 40));
          }
        }
      }
    };

    recognition.onend = () => {
      if (voiceActive) {
        blog('SpeechRecognition onend → auto restart');
        try { recognition.start(); } catch (_) { /* 忽略 */ }
      }
    };

    recognition.onerror = (e: any) => {
      blog('SpeechRecognition error:', e.error, e.message || '');
      if (e.error === 'not-allowed' || e.error === 'service-not-allowed') {
        voiceActive = false;
        recognition = null;
        blog('麥克風被拒絕，語音停用');
      }
    };

    recognition.start();
    blog('SpeechRecognition started (zh-TW)');
  } catch (err) {
    blog('⚠ SpeechRecognition 建立失敗', err);
    recognition = null;
    voiceActive = false;
  }
}
```

**重點**：
- `showMicPermissionOverlay` 的 allowBtn click → `getUserMedia` → 有效 user gesture → Chrome 會彈標準的麥克風授權提示 → 使用者允許後 → `initSpeechRecognition` 啟動語音
- 已授權的站（`result.state === 'granted'`）直接跳過 overlay，無感啟動
- 「跳過」按鈕讓使用者可以不錄語音，不阻擋 DOM/console/network
- overlay 用 `z-index: 2147483647` 確保最上層、`position: fixed` 黏在頁面頂部
- overlay 只在錄製開始時出現、授權後或跳過後立刻消失
- **注意 TypeScript 型別**：延續 DONE-08 的做法，用最小 interface 取代 `any`（範例用 `any` 只是示意，Code 請自行用既有的 `SRInstance` 等型別）

#### §3 Workers API 程式碼（`server/src/index.ts`）

兩個端點：

**POST /api/reports** — 接收 RecordingPayload，存 R2 + Supabase
```
Request body: RecordingPayload JSON（跟擴充匯出的一模一樣）
流程：
1. 產生 report_id（crypto.randomUUID）
2. 從 payload 拆分：rrwebEvents 存 R2（大檔，可能幾 MB）、其餘 metadata 存 Supabase
3. R2 key: `reports/${report_id}/rrweb.json`
4. Supabase insert into reports:
   - report_id, url, title, browser, screen_size
   - console_count, network_count, voice_count, rrweb_count
   - rrweb_r2_key（R2 的 key，之後讀取用）
   - console_logs（JSONB，直接存）
   - network_errors（JSONB，直接存）
   - voice_transcript（JSONB，直接存）
   - created_at
5. 回傳: { report_id, share_url: `https://bugezy-api.<account>.workers.dev/report/${report_id}` }
```

**GET /api/reports/:id** — 讀取報告
```
流程：
1. 從 Supabase 讀 report metadata
2. 如果要 rrweb 資料：從 R2 讀 rrweb.json（或回傳 presigned URL）
3. 回傳完整報告 JSON
Response: {
  report_id, url, title, browser, screen_size,
  consoleLogs, networkErrors, voiceTranscript,
  rrwebEvents: [...] 或 rrweb_url
  created_at
}
```

**CORS**：加 `Access-Control-Allow-Origin: *`（MVP 先全開，第 5 代再收緊）
